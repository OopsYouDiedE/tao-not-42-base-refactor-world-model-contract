# Step 1:TwoTower × 连续动作控制 可行性实验

**上游**: [[fovea-twotower-design]] §5 闸门实验的最小化落地
**原则**: 只验证无先例背书的环节;有背书的环节(流匹配动作头=π0,冻结配方=Table 2)降为烟测。
**平台**: 本机 L4 23GB(训练);运行时延迟指标仍按 3090 记账。

## 1. 待证命题(可证伪形式)

- **S1(烟测)**: 流匹配块去噪能拟合人类连续键鼠 chunk。
- **S2(=Gate A)**: 冻结 Context 塔的 Mamba 状态里存有控制所需的历史信息。
- **S3(=Gate B/C, 核心)**: "读冻结世界模型的状态"这条通道有效——
  播种 > 无播种;冻结 ≥ 联合微调(允许小差距)。
  **这是全设计唯一无外部先例的环节,本实验的存在理由。**

## 2. 最小模型(每塔 ~15-25M)

- d_model 384, 12 层 = 9×GatedDeltaNet/Mamba-2 + 3×GQA(每4层1个), 与 v0 设计同 4:1 配比。
- 视觉 token: 冻结 DINOv2-S 吃 160×90 缩放帧 → ~66 token/帧(零训练 tokenizer,对齐
  dreamer4 连续 DINO token 习语;凹式前端**不进本实验**——变量隔离)。
- 动作 token: 22 维(dx,dy 对数尺度 + 20 键 multihot, 词表=dataset.KEY_NAMES) → 1 token/帧。
- 线性注意力内核: `pip install flash-linear-attention`(纯 triton,免编译;与 Qwen3.5 同源)。
- 复用: `net/dreamer4/dynamics.py::ShortcutHead`(流匹配速度头)、动作 AdaLN、
  `train/gaming500/dataset.py`(seq 窗口 + unpack_keys)。

## 3. 数据

- 单游戏切片: the-finals ~10-20h → `tests/encode_gaming500_hdf5.py` 编码为分片。
- 10Hz 重采样;按 clip 切 80/10/10,同一录像不跨集。

## 4. 训练与消融矩阵

| 编号 | 配置 | 验证什么 |
|---|---|---|
| R1 | Context 塔预训练: 因果交错流[66视觉+1动作]/帧, 目标=下帧DINO latent MSE; 训完冻结 | 世界模型本身 |
| PA | 探针: 冻结R1各层Mamba状态 → 线性头预测未来0.5s(是否开火, 鼠标速度档) vs **两个基线**: 单帧同型探针 + 单帧⊕裸动作历史(16帧)探针——后者是动作泄漏对照(efference copy 使按键自相关可冒充世界记忆) | S2 |
| R2 | Action 塔: 同源初始化, 播种R1状态 + cross-attn当前帧66 token, ShortcutHead 去噪 H=8拍×22维 chunk | 主案 |
| B0 | 当前帧 MLP(无历史无塔) | 历史是否 matter |
| B1 | R2 去播种(零初始状态); 与 R2 固定同一 RNG 完全配对(init/数据顺序/ε 全同) | S3-播种收益 |
| B1.5(R2>B1 后) | R2 错配播种(batch 内 roll,拿别人历史) | 播种收益=内容还是统计先验 |
| B2 | R2 不冻结(联合微调两塔) | S3-冻结代价 |
| B3(可选) | 单塔 AR 离散动作头 | 块去噪 vs AR |

探针标签全部免费来自操作日志(未来开火/鼠标速度),不需要人工或 OCR 标注。

## 5. 指标与通过阈值

离线(测试集):
- 鼠标: dx/dy R²(对数域)、方向命中率;按键: per-key F1(重点 key_attack)。
- 反应性: 取"敌情出现"片段(YOLO/老师标注定位),预测开火时刻相对人类的提前/滞后分布。
- chunk 内平滑度对齐人类(块一致性)。

| 命题 | 通过 | 失败处置 |
|---|---|---|
| S2(历史) | PA(状态探针) AUC ≥ 单帧探针 +5pt | 世界目标没把控制信息压进状态 → **先升输入分辨率(凹区 center 口径)重试**;仍败才加辅助目标,且辅助目标**必须 ≠ 探针目标**(否则探针平凡通过,闸门被掏空——循环论证) |
| S2(世界) | PA AUC ≥ 单帧⊕动作历史探针 +2pt;**并行判 STATE 口径**: AUC(STATE⊕帧⊕动作) ≥ AUC(帧⊕动作) +2pt(STATE=9层recurrent_state沿K均池,即 R2 播种真实递交的通道;末层 hidden 均池会稀释单帧信息,见 90p 诊断) | 状态≈动作缓冲区,世界塔必要性存疑 → S3 仍可跑,但架构叙事从"读世界模型"降级为"读历史摘要",1.5B 塔的规模论证重审 |
| S3a | R2 > B1 显著(主指标=**f1_attack**,≥+10%相对;f1_keys/r2_mouse 报告不判定)。**预登记后备**(2026-07-04,f1_attack 在 6000 步内贴地<0.1 时启用):frame 级 attack **连续输出 ROC-AUC**,窗口级配对 bootstrap,ΔAUC 95%CI 下界>0 为过;过关后 B1.5 错配 ≪ R2 方可归因于历史内容 | 播种通道无效 → 退回"共享近窗KV"方案(dreamer4 原映射) |
| S3b | R2 ≥ 0.9×B2 | 冻结代价过大 → 部分解冻(顶部N层)扫描 |
| S1 | R2 达 B2 的 90%+ 且开火 F1 > B0 明显 | 动作表征问题, 先修 log 尺度/迟滞再议架构 |

**离线 BC 指标的已知局限**: 复合误差不体现。闭环留作 Step 1.5——
用仓库 godot_rl/crafter 造连续鼠标瞄准小任务, 脚本专家生成数据, rollout 测成功率;
不阻塞 Step 1 判定。

## 5.5 第一轮结果(2026-07-04, 90p 全图口径)—— Gate A 双 FAIL,归因=分辨率×目标函数

58.6M 塔 / valorant 24h / R1 6000步 bs4 收敛(eval MSE 0.272→0.185 平台)。
探针(`runs/ftt_r1/gate_a.json` + 诊断 `gate_a_diag.json`):

- attack: FULL 0.544 / FRAME 0.561 / FRAME+ACT 0.711 → 双判据 FAIL;
- 诊断三排除:MLP 不优于线性(读出方式无罪);onset 标签下 ACT 掉到 0.519
  (0.711 全是按键自相关——泄漏对照按设计起效);FULL⊕ACT ≤ FRAME+ACT(状态零边际);
- 但 onset 下 FRAME=0.62(视觉有真信号)、motion FULL 0.695≈FRAME+ACT(状态确实压缩了
  相机动力学)→ 初判病灶=下帧 DINO-MSE 在 90p 被相机运动主导,敌人只占几像素。
- **二轮反转(gate_a_v2.json,同一 90p ckpt,探针改探真实播种通道)**:
  STATE(9层recurrent_state)单独 0.676 ≫ FRAME 0.609;STATE⊕帧⊕动作 0.780 vs
  帧⊕动作 0.737 → **verdict_world_state PASS (Δ+4.3pt)**。
  首轮 FAIL 大半是**探针接口伪影**——末层 hidden 均池稀释了状态内容;状态在 90p
  就已持有超出动作缓冲的世界记忆。教训:闸门必须探被下游实际消费的通道。
  (注意:同 ckpt 两次探针 FULL 差 ±0.05-0.07,采样噪声大;STATE 的 +4.3pt 是
  同一采样内配对差,相对可信,360p 复测确认中。)
- 处置:S3a(R2 vs B1)在 90p 塔上即刻开跑(状态确有内容,播种消融已有意义);
  并行 360p 存储 + center 原生 126×126 裁剪(=凹区最小实现,敌人 ~2.5px→~10px)
  **目标函数不动**重训 R1 复测天花板。辅助目标留作二线,警惕循环论证。

**S3a 90p 结果(`runs/ftt_s3a_90.json`,RNG 配对 + 固定采样噪声 + 窗口级 bootstrap)**:

- **机制成立**:keys 宏 AUC R2 0.595 vs B1 0.546,Δ+0.048 **CI95 [+0.028,+0.089] 显著**;
  f1_keys 全程 R2≫B1(终点 0.484 vs 0.381,+27%)。播种通道确实向动作塔传递了可用历史。
- **attack 判据 FAIL 但无检力**:Δauc +0.018 CI 含 0——两臂都贴随机(0.558/0.540,
  onset 双双 ≈0.535),90p 视觉底料无敌情信号(与 GateA FRAME=0.61 天花板一致),
  差值检验在地板上没有意义 → 按预登记分辨率规则**不触发"退回共享近窗KV"**,360p 复测定夺。
- 待解:keys 收益是世界记忆还是动作缓冲(efference copy)——360p 轮 attack 有检力后,
  连同 B1.5 错配对照一起回答。

**GateA-360(`runs/ftt_r1_360/gate_a.json`,center 凹区口径,n_test=2000)**:

- FRAME 0.647(90p 0.609)——凹区确实抬高视觉天花板;STATE 单独 0.681;
- verdict_world_state **擦线 FAIL:Δ+1.98pt vs 阈值 2pt**(90p 为 +4.3pt PASS;
  两轮方向一致);verdict_history FAIL(+1.6pt)。
- 处置:提前停被人工放行——该闸门的设立前提("状态无内容则 R2 无意义")已被
  90p S3a 的 keys 显著性推翻,S3a 本体才是终审;GateA-360 如实记为擦线 FAIL,
  不改判、不动阈值。R2/B1/B1.5 三塔照跑。

## 5.6 终审(2026-07-05,`runs/ftt_s3a_360.json`,n=2000 窗,三方 RNG 配对)

| 判据 | 90p | 360p 凹区 | 判定 |
|---|---|---|---|
| 播种传历史(keys ΔAUC R2−B1) | +0.048 CI[+.028,+.089] | +0.063 CI[+.035,+.087] | **PASS,两轮显著** |
| 内容归因(keys ΔAUC R2−B1.5) | — | +0.045 CI[+.024,+.074] | **PASS,错配≈零状态** |
| 世界记忆(attack ΔAUC R2−B1) | +0.018 CI含0(两臂贴地) | +0.007 CI[−.011,+.024](两臂0.61,有检力) | **FAIL,真空结果** |

**结论**:
1. **机制成立**——O(1) 状态播种作为唯一历史通道可用:传递本局特定历史
   (f1_keys R2 0.474 vs B1 0.287 vs B1.5 0.286),冻结+同源初始化+流匹配全链路稳定。
2. **独特卖点未兑现**——状态超出动作缓冲的世界记忆(敌情预判)在动作塔输出端≈0,
   360p 有检力下的真空。按 §5 预登记:叙事降级"读世界模型"→"读历史摘要",
   **1.5B 规模论证暂缺实证支撑**。
3. **定性边界**:本实验只证伪"纯下帧MSE目标 + 58M + 离线BC"组合,未测完整 P1
   (27B 语义蒸馏 NLL——与探针目标不同,无循环论证,是干净的下一级);
   onset 双臂 0.58-0.59 提示离线 BC 指标对反应性收益可能天然钝感(→Step 1.5 闭环)。
4. **判定缺口(下一个必做,半小时级)**:B0.5 = 当前帧+裸动作token(无塔无播种)。
   若追平播种的 keys 优势,则播种通道本身在部署上不必要;不追平,播种才有独立价值。
   在 1.5B 上押注前必须先跑它。

## 6. 排期(L4)

数据编码 ~1天(带宽决定) → R1 一晚 → PA 半天 → R2/B0/B1/B2 各 4-6h(可串行两天)。
**一周内 S1-S3 全部出数。**
