# 设计：学习式注意——让模型自己决定看哪里、看哪帧

> 2026-07-02。回答"能否自动学习到要在模型里学会注意"：1080p 塞不全，但也不该
> 粗暴降分辨率 + 均匀 skip-frame 了事。前置阅读：[design_gaming500_hd_pretrain.md]
> 的三道闸分析（§7）——本设计是闸 1（下采样）与时间轴的学习式解法。

## 0. 核心原则：amortized saliency（事后回归，绕开不可微选择）

硬注意（选区域/选帧）不可微，经典解法（REINFORCE/Gumbel）高方差、难训。
世界模型场景有捷径：**训练中每步都真实产出"哪里预测得差"**（逐 token 流损失、
gen−persist 逐区域差）。因此显著性头只需做监督回归——"预测全局低清流在各区域将
产生的误差"——选择本身（top-k 硬裁剪、优先采帧）放在数据侧，不需要梯度穿过。
训练稳定，额外代价接近零，且老师随模型进步自动更新。

## 1. 时间轴：学习式挑帧（v0，零新参数）

- 现状：`--motion_sample 4` 锦标赛，准则=像素帧差能量（手工）。
- 升级：**准则换成模型自身分数**——候选序列的流损失，或难度头 D(ctx) 输出。
  即 prioritized experience replay 的世界模型版：战斗/挖掘/视角剧变密集采样，
  平移赶路稀疏化；帧预算按"模型还没学会的程度"分配，替代均匀 skip-frame。
- token cache 协同：分数是训练副产品，每 epoch 重算优先级几乎免费。
- **前置条件**：难度头当老师前先修 IG 恒负异常；修好前用流损失分数替代。
- 代价：几十行（锦标赛准则函数换成模型打分回调）。

## 2. 空间轴：双流凹式（foveated）tokenizer（v2，架构级）

- **全局流**：128² 全景 → 16 token（现架构不动），负责布局与大尺度动力学。
- **saliency head**：几层 conv 读全局编码器特征图 → 误差热图 → top-k（k=2~4）。
- **凹式流**：按热图从 **1080p 原生像素**裁 k 个 128²/192² 窗口（不缩放）→
  同一编码器 → k 组凹式 token，附 (x,y,scale)-MLP 位置编码（复用动态分辨率
  讨论的 pos_spatial 解耦方案）→ 与全局 16 token 拼接进 dynamics。
- **监督**：回归"全局流单独预测时的逐区域实际误差"。准星处方块开裂、快捷栏
  变化、远处实体运动等全局流搞不定的区域自动成为注视点。
- 代价：k=2 时 ≈ 2× 编码 FLOPs；token 16→16+2k，temporal attn 增量平方项仍小。

## 3. 存储矛盾与解法：凹式流需要 1080p 在场，磁盘留不住 1080p

1. **v0 转换期（手工显著性）**：裁剪流放置从纯随机 → 运动能量加权，转换时把
   高频活动区裁下存好。零模型依赖，先拿大头收益。
2. **v1 学习期（在线流式）**：saliency head 就绪后，对精选子集（几十小时）
   重新从 HF 流式下载 1080p → NVDEC 在线解码（实测 780 帧/s、零 CPU）→
   按热图在线裁剪 → 不落盘。付网络+一次解码，磁盘零占用。

## 4. 路线与自我约束

| 阶段 | 内容 | 代价 | 状态 |
|---|---|---|---|
| v0 | 锦标赛准则→流损失分数；转换期运动能量加权裁剪 | 几十行 | 进下一轮消融矩阵 |
| v1 | 修 IG/难度头 → 温度化 prioritized 帧采样 | 诊断+小改 | 待 IG 诊断 |
| v2 | saliency head + 双流凹式 token + 位置编码 | 架构级 | 设计入库，按需启动 |

主线是 mine_stone（[[project-goal-mine-stone]]）：v0 预计拿走本方向 60-70% 收益；
v2 放在多游戏预训练验证 128² 基线之后启动，避免一次引入两个变量。

## 5. 风险

- 难度头老师不可信（IG 恒负未诊断）→ v0 一律先用流损失分数。
- prioritized 采样引入分布偏移（只看难的→易段遗忘）→ 保留 ε 比例均匀采样兜底。
- 凹式 token 打破"16 token 固定网格"假设，dynamics 的 pos_spatial 与 decoder
  flat Linear 两处耦合点需先解耦（与动态分辨率工作共用改造）。
