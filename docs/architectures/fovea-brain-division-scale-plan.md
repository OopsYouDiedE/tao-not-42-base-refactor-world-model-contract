## 2. 慢脑↔快脑接线(运行时回路)

```
慢脑 LLM(低频/事件触发)                     快脑(每帧 ~15-20Hz)
  读: 文本消息总线                            读: 统一头 token [K, 几何6+类分数]
      = token流符号摘要 + 背包变化                + goal 相对折叠(p_goal, p_other)
      + 注册表条目 + 任务书                       + 上一步动作
  写: ①感知点名: 新类文本→PE→向量bank         写: 相机 mu-law bins + 按键
        (快脑立即"看见"新类别)              goal 在两次慢脑输出间零阶保持
      ②策略指令: 指令→goal(类别/向量)
      ③注册表读写请求(结构化)
显式注册表(黑板,事件驱动): 方块/途经点世界坐标条目;age/bearing 用算术;
  raycast+位姿自动登记;"回家" = 注册表回放为导航 goal 序列
```

关键机制(均已代码落地):goal 相对 token 折叠使快头**结构上类无关**——"听指挥"
不靠 goal 向量语义,靠输入契约,新类别零重训;慢脑文本经①改写"看见什么"、
经②改写"追什么",都是 O(1) 向量插拔。

## 3. 大规模训练配方(三条全自动数据引擎)

**原则**:预训练模型不改行(冻结骨干+外挂小头),监督全部程序生成(零人工标注),
判据先于训练登记,凡"记忆"优先显式存储。

### 引擎 A:规则引擎 → 慢脑知识(E2 路线)
- 科技树/配方/工具依赖是可执行规则 → 组合生成 (库存状态, 差额计划) 对,含
  <think>反向依赖推理</think>;多解拓扑校验器判正确(不逐字对答案)。
- 规模化:扩全物品科技树(数百物品)、加干扰库存、多措辞模板;样本无限,瓶颈只在
  LoRA 步数。**这就是"最快让慢脑学会 MC 知识"的答案:喂规则引擎生成的(状态,决策)对,
  不是喂百科文本。**
- 升级路径:1.5B 验证 → Nemotron-4B(其语言侧;递归状态不再委以视觉记忆)。

### 引擎 B:GT 工厂 → 感知校准(G1 路线,已验收)
- setblock 已知坐标 + raycast 锚定 + 位姿投影 → 免费分割 GT;
- 新类别流水线:文本 PE 冷启动(粗召回)→ 采域内格嵌入 → 学习向量(逻辑回归级)
  → 需要精确掩膜再训该类 conv 头通道;骨干永冻结。
- 规模化:课程随机化(布局/距离/光照/生物群系),每类几百帧即够(G1:25 局)。

### 引擎 C:示范工厂 → 快脑技能(E1 路线)
- 教师=位姿投影瞄准/raycast 脚本,但**监督落在 token 可表达量上**(目标 token 居中/
  box 变大)——动作必须是学生输入的函数(mamba_seed 教训的结构化落实);
- **教师还必须是确定性的(或潜变量可观测)**:v12 发现,教师搜索方向掷硬币
  (rng.choice ±1)时,同观测状态的标签 ±15° 对冲,BC 条件均值=0 → 学生切换率
  钉死 0.17 而其余指标全达教师水平。随机教师策略在潜变量不可观测处不可 BC——
  确定化(恒右转)或把潜变量放进观测;
- 示范分布要覆盖闭环状态:开局扰乱 + 局中多次指令切换(v1 单切换→切换率 0.25 的教训);
- 规模化:技能=goal 类型×课程随机化;BC 起步,ReST/GRPO(raycast 距离/居中做 reward)精修;
  软渲染 ~5s/局,千局级示范一夜可采。

### 训练顺序(依赖链)
1. 引擎 B 校准感知(每新环境/新类一次,分钟级)→ 2. 引擎 C 练执行头(每技能 3k 步,
分钟级)→ 3. 引擎 A 持续注入慢脑知识(小时级)→ 4. 注册表纯工程 → 5. 全回路闭环评测
(指令链任务:慢脑出计划→逐条驱动快脑→核查库存→回家)。
可选:从零 GDN 塔播种(若 B0.5 证独立价值)作为快头附加历史通道。

## 4. 本轮验证记录(E1/E2)

### E1 指令闭环(judge=活环境四臂:student/teacher/frozen/random,同 RNG 同课程)
- teacher 天花板:0.1°/切换 1.0/到达 1.0(oracle 与评测管线双验证);
- v1(41 局单切换示范):离线 cam_acc 0.699 vs 持续 0.593(+10.6pt;旧未校准 token
  +6.7pt → **E3 顺带回答:校准 token 确实抬 BC**);闭环追踪 2.1° 好,
  **切换 0.25 败**——示范只含 41 个切换事件且开局白送对准;
- v2(50 局,开局扰乱+每 40±10 步切换):离线 0.684/持续 0.557,但闭环学生
  **退化成 frozen**(p1 与 frozen 逐位相同=完全不动相机)——**惯性坍塌**:
  锁定期小箱压倒多数 → argmax 收敛"永远微调";prev_action 因果混淆(看见自己的
  noop→继续 noop)。v1 的"追踪 2.1°"由此重判:主要是开局白送对准的假象。
- v3(bin 逆频权重+prev_dropout 0.5):翻到另一失败面——**过度转身**(39°/73°,甩过目标)。
  权重无帽矫枉过正。→ 定论:**纯离线 BC 到不了闭环,示范工厂必须带 DAgger 环**
  (教师是环境内免费 oracle,学生开车教师打标签,DAgger 几乎零成本);
- v4/v5(权重加帽 3× + DAgger 一轮):闭环仍未过(71°/84°)。DAgger 救不了,因为——
- **v6 根因(本轮 E1 最重要发现,两层)**:
  ① 位姿投影教师在目标出视野时仍能直转目标 = **特权教师的隐蔽复发**(mamba_seed
  教训):重获取行为从 token 上不可学。修法=观测一致 TokenTeacher(只消费学生同款
  token,可见瞄 token 中心/不可见匀速搜索),可学性由构造保证。**铁律:教师必须是
  学生观测的函数,位姿/raycast 只许用于评测**。
  ② TokenTeacher 一上就暴露 **pf 提案池化命名假阳性泛滥**(教师锁灰墙假铁,err 89°)
  ——与 G1 量化一致(提案并集 0.15 vs conv 稠密 0.53)。修法=token 从 conv 分割头
  连通域构建(TokenHead v2)。修后教师段末锁定 p25=8.5°(感知对时控制器成立),
  但锁定率 0.38,按类分解:iron 0.17/coal 0.33/dirt 0.56——**铁类假阳性是唯一残余
  瓶颈**(灰石墙远距斜角冒充铁矿)。
- **E1 现状与已定路径**:未收口。下一步(引擎 B 常规扩产,数小时级):困难负样本
  课程(纯石墙×距离×角度)加练 conv 头 → 教师锁定率 ≥0.6 → 采 demo+BC+DAgger 一轮
  → 闭环三判据复评。E3 已答:校准 token 抬离线 BC(+10.6pt vs 旧 +6.7pt)。
- **续训(用户令 GPU 不停,链 1–3)**:
  · 困难负样本(12 局纯石墙)一次性把铁假阳性 177px/帧 → **0**(精确-召回跷跷板:
    铁召回同时塌 0)→ 远墙正样本(rand2)进训练 + 教师选择器改"面积×概率"后,

    (同观测天花板),oracle 只作环境上限**;
  · v7 闭环(70 示范+30 DAgger,conv v3 token):**学生首次真正听指挥**——
    p1/p2=17.9°/13.3°(教师 10.8/10.8,frozen 24/16.3),切换后误差已逼近教师;
    切换率 0.36(教师 0.78)欠"果断发起搜索",DAgger 对症;到达判据发现标定错误

  · 链 4(v9,n=13):学生追踪已达教师水平(p1 15.3° vs 教师 16.8°),切换 0.23/到达 0.0

## 4.5 上集群(高性能设备)的就绪判据(2026-07-07 深夜登记)

**总原则:集群只买"梯度×规模",买不到接口/数据/设计修复——上集群的标准 =
"剩余唯一瓶颈已被证明是算力"。** 五道闸门,全绿才许烧钱:

- **C1 系统闸门**:单机全回路先通。①E1 三判据 PASS(学生≤token教师×1.5/切换≥0.7×教师/
  到达≥0.6,至少一个技能族);②指令链端到端原型:慢脑计划→goal→快头执行→库存核查→
  注册表回家,三步链成功率 ≥0.5(判据今登记;**当前最大缺口,从未跑过**)。
- **C2 规模信号闸门**:每个想放大的组件先画 ≥3 个小尺度的单调未饱和曲线
  (快头 2M/8M/20M;示范 100/500/2000 局;慢脑 0.5B/1.5B/4B)。W4 教训的延伸:
  曲线平了或掉头的组件不准上集群。
- **C3 数据引擎闸门**:多环境并行采样吞吐实测线性;自动质检(schema+教师指标阈值
  自动拒收);数据版本化+分布报告。
- **C4 工程闸门**:一条命令复现单机最优(种子固定);断点续训;评测-训练解耦(已成立);
  梯度累积≟大batch 对拍。
- **C5 纪律闸门**:集群任务的成功判据**与止损判据**(如 20% 预算处低于 X 即中止)
  先于启动登记。

现状:C5✅ C4🔶 **C1✅(2026-07-08,executor=teacher 口径)** C2🔶 C3🔶。

## 4.6 设备速度鲁棒性论证(2026-07-08,三探针实测)

**问题拆解**:输入是固定帧率假设还是可变帧率预期?对慢设备鲁棒吗?

契约审计:
- **快塔=固定帧率假设**(两处硬编码:动作语义"每 tick 相机增量"、rel token 无 dt);
- **慢塔流契约预留了可变帧率**(act_featurize 带 dt/30 特征,gaming500 真人变帧率
  数据重采样 10Hz+dt),但 craftground 数据 dt≡2 常数,能力未被训练激活;
- 环境锁步(一 step=一 tick)⇒ 固定 Δt 由平台构造性保证——快塔假设至今未咬人的原因。

三探针实测(v17 学生+token 教师,同种子对照,runs/speed_*.json):
1. **墙钟不变性 ✅**:每步注入 300ms(≈10 倍慢设备)全指标在噪声带内
   (学生 15.0→16.4°/教师 9.3→10.2°)——锁步下训练/评测对设备速度完全鲁棒;
2. **时间尺度敏感度 ✅**:action-repeat=2(等效帧率减半)无坍塌(学生 14.7°/0.75 到达)
   ——每观测重瞄 token 中心的反馈结构天然耐率变;
3. **延迟预算**:统一头 23ms + 22M 快头 5ms = 28ms/帧 → 36Hz(3090),对 MC 20Hz
   tick 有 2.4× 余量;慢塔秒级延迟经 goal 零阶保持异步化,不占帧预算。

实时部署(非锁步)升级阶梯(登记,不紧迫):(a) 动作改速率语义(度/秒×实测 Δt,
执行器层零重训);(b) dt 进快头输入+示范随机跳帧增广(补齐慢塔预留的 dt 通道);
(c) 固定节拍执行器。
