### 1.1 O1 吞吐分析(结论:预编码慢塔 latent 被否决,真 batch 是本机唯一杠杆)

- **DINO 现场编码占比可忽略**:dinov2-vits14 仅 22M(vs Nemotron 4B),bs1×seq32=32 帧
  前向 ~O(10ms),对 3.8s/步占比 <1%。→ **慢塔 patch-latent 预存(~22GB)预期收益 <5%,
  不做**(此前"预存换吞吐"设想被算术否决)。
- **真 batch**:`batch_to_stream` 支持 [B,L](R1 曾 bs=8);4B+梯度检查点下激活随 B 增长
  有限,VRAM 21.0/24.6G 为约束。已加 `--bs`。预期 bs2(accum2 等效批不变)提升 1.3–1.8×。
- **本机之外**:seq32→更长或 3 万步级别的训练,3090 无解,是换卡问题不是软件债。
  软件侧把 O2(续训)做好即可随时迁移。

### 1.2 O4 相机尺度校准

h5 动作按 30Hz 事件帧存,图像 15Hz → 区间聚合使 |dx| 分布约为 30Hz 逐帧的 2×,
CAMERA_SCALE=10 截断比例会升高。处置:encode_g500_feats **归一 action 与原始 dx_px 双存**
(改 scale 免重编码),逐段打印 |dx| p50/p99 与 clip%;若 clip% 中位 >15%,快头训练
换 scale=20(需重算 action 或训练侧现场归一,dx_px 已备)。
