# 07-08 路线假设验证记录

对应定案见记忆 fovea-route-decisions-2026-07-08。验证原则:能离线验的先离线验,
每条假设给"证实/证伪/部分/排队"四态 + 复现命令。

## ✅ V1 稀疏键假设(证实)

**假设**:E/Q/热键栏类稀疏键在示范分布里不可学(归慢塔);模式掩码可从数据提取。
**实测**(41 track + 29 wood + 40 craft + 24 iron 示范,~21k tick):
- `inventory(E)` 仅 craft 集出现,正例率 1.1%(≈每局一次);其余集严格为 0。
- craft(GUI 段)WASD/space/sprint 支撑为 0;`sneak` 35.6% —— Shift 作 GUI
  修饰键被数据独立证实(外部方案手写掩码表在热键栏/Q 上则与 MC 机制冲突,
  待 GUI 页数据仲裁)。
**结论**:稀疏键归慢塔动作空间正确;数据驱动掩码(按 (模式,键) 支撑阈值)可行。

## ⚠️ V2 文本 PE↔校准向量几何(部分——重要负发现)

**假设**:小 MLP/Transformer 可把指令文本映射到 YOLOE 校准级编码向量。
**实测**:g1 校准向量与对应类文本 PE 余弦 ~0(对角 -0.005 vs 非对角 0.001),
范数 41–49 vs 单位球。原因:校准打分走 `norm(BN(emb))@W.T+b`,**W 活在 BN 特征
基,PE 活在原始嵌入基,两基近正交**。
**修正**:适配器不能是"PE+小残差",必须学完整换基线性映射;当前仅 3 对样本,
欠定不可判。**解锁条件:校准类对数增长**(log 类是第 4 对,wood_gt 已采)。
架构文档"同空间互换"表述应理解为"可插同一槽位",非几何同向。

## ✅ V3 心跳延迟预算(证实,1.5B 现役底座)

- 静态前缀(配方卡 684 tok)一次性 prefill:349ms
- **每心跳 85ms**(36 tok 状态行增量 prefill + 3 tok 微决策,KV cache 复用)
  → 1Hz 占卡 8.5%,与 YOLOE+快头共存成立
- 全量重规划 3.6s(169 tok)→ 证实必须稀疏触发+零阶保持
- 显存 3.1GB。复现:见本文档同名 session 的 bench 脚本(逻辑内联在对话)。

## ✅ V4 I_gui 检测器(机制修订后证实)

## ✅ V6 IPM 地平面投影(数学内核精确)

wood_gt 34 局、513 个可见树干底块样本:已知位姿+贴地接触点下,前向投影→IPM
逆投影的最近角点误差 **0.0000 格(全距离档 0–25 格)**。
真实管线误差将来自掩膜底边噪声+位姿脚手架,不来自投影数学。
"32 原型掩膜→IPM→EgoMapNorthLoc" 几何通路成立;高于地面的目标(墙上矿)
违反平地假设,需深度通道(Y2 结论不变,二期)。

## ✅ V5 Qwen2-VL-2B 心跳预算(证实,A2 底座绿灯)

3090 bf16,帧+状态行+4 token 微决策,每拍全量 prefill(无前缀缓存,保守口径):
- **448×256:94ms/拍(192 tok)**;336×192:82ms;224×126:76ms;显存 4.2GB。
- → 448×256@1Hz 占卡 ~9%,2Hz 亦可行;与 YOLOE+快头共存余量 ~17GB。
- 生产形态加静态前缀 KV cache 后边际成本更低。A2 换件本机可行性成立。
