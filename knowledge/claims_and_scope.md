# 期望与现实校准:本项目实际在证明什么

> **文档性质**:不讲架构(归 [mental_world.md](mental_world.md))、不讲外部对照(归 [world_model_landscape.md](world_model_landscape.md))、不记录逐轮 run 数(属历史活动,归 git log 与项目记忆)。
> 本文只做一件事:把**当初的目标**、**系统当前能诚实主张的命题**、**两者的差距**对齐,作为"是否值得继续投入"的判断依据。
> 写它的原因:研发过程容易从大目标滑向一个代理任务、再滑向代理的子角落(本项目曾连续数轮在 onset 单指标上打磨);需要一个定期回头核对"我们到底在证明什么"的地方。

---

## 1. 两层目标:不要把元目标当成操作命题

- **元目标(meta)**:AI 像动物一样,自主从场景交互里发现因果,不依赖大规模预训练。
- **操作目标**([mental_world.md](mental_world.md) §6):看一段视频 + 试跑几步,**权重冻结**下 in-context 适应新玩法。

操作目标是元目标在"可证伪 + 能在现有算力上测"约束下的**收窄版本**,经两步缩减:(a) 把"发现因果"收窄为"读出 / 在线适应 action→effect";(b) 把"不依赖预训练"这一约束暂时放掉(见 §2)。**认清这是收窄而非等价,是校准的第一步**——否则任何具体指标都会显得"离目标太远",因为它本就只覆盖目标的一个子集。

---

## 2. 直面预训练问题:我们没有、也(暂时)不打算"从零学感知"

- **事实**:感知使用冻结 DINOv2/v3(海量图像预训练),控制语义使用 MiniLM(文本预训练)。元目标字面上的"不依赖庞大预训练",当前系统**不满足**。
- **为何这样选**(已在 [mental_world.md](mental_world.md) §3 记录):从零随机骨干无先验、holdout 退化(prime-leaf-7 实测);冻结预训练骨干给 JEPA target 一个独立于本任务数据的意义来源,是 JEPA 的前提。现有算力下也训不出可用的感知。
- **重新定位**:动物不是白板。视觉皮层不是一生学出来的,而是进化"预训练"好、出生即带的先天结构。把冻结骨干类比为"**先天的视觉皮层**",则"冻结感知先验 + 从交互学当前环境的动力学 + 在线重绑定"比"一切从零"**更**接近动物的学习方式——动物一生学的也正是这个环境里的因果,不是重学视觉。
- **结论**:把元目标的硬约束从"零预训练"修正为——**感知先验可借(类比先天皮层),但因果 / 动力学必须从 agent 自身交互里学、且能在线适应**。在这条修正后的标准下,系统是对齐的;"用了 DINOv2"不再算作偏离,后续不必为此纠结。

---

## 3. 系统当前能诚实主张的命题(claim 边界)

分三档。逐轮具体数字见项目记忆,本节只给命题与其证据类型。

### 可以主张(已有实验支持)
- 给定冻结感知,小模型能在潜空间学出**以动作为条件的 Δz 动力学**,胜过 persistence(预测 0)基线(归一化基线 1.0,运动样本预测稳定低于基线)。
- 从潜变化能**读出动作**(逆动力学):键盘平衡准确率、鼠标方向均显著高于随机基线 ⇒ **动作信息确实进了潜表征**,满足 [mental_world.md](mental_world.md) §9 的抗作弊口径(开环对照、结构化 query 读出)。
- **在线重绑定有信号**:控制映射被逐 episode 打乱后,模型在一段 episode 内靠 context(非权重)部分恢复"动作→潜变化"映射,且适应随 context 增长(late > early)。这是**最接近元目标**"从交互发现因果"的一条,也是操作目标的核心证据。

### 部分证据 / 有结构性上限
- 键盘 **onset**(刚按下的首区间)是**校准上限**问题,不是 bug:首区间证据最弱,良好校准的后验本就被拉到 0.5 以下(实测中位锁在 ~0.37–0.38,**与初始化无关**),故工作点是 **θ≈0.2** 而非 0.5。这条已诊断到底、结案。它衡量的是"读出头能否认出按了哪键",**不是**"模型是否发现因果"——不应再当主线打磨。
- 多步开环 rollout 能在若干步内胜过"原地不动",但预测保真度随深度衰减:前向想象**能撑一小段、非长程**。

### 不能主张(范围之外)
- "从零、不依赖大预训练,像动物一样发现因果"——感知是借来的(§2)。
- "模型理解 Minecraft 因果 / 物理"——证的是潜表征里有可读的动作效应、可在线重绑定的映射,**不是**因果世界知识。
- "onset 高低 = 因果发现强弱"——onset 是读出校准指标,受本节上限约束。
- 任何来自真实交互环境的主张(通关率、主动探索)——当前全是离线 VPT 录像,主动交互轴未测([mental_world.md](mental_world.md) §10)。

## 4. 哪条线朝元目标走,哪条已到上限

- **朝元目标**:**in-context 因果发现 / 重绑定**(§3 第三条 + [mental_world.md](mental_world.md) §6)。把它做扎实——更多键、训练没见过的重映射 / 效果、拉长适应曲线、加 baselines(无记忆 / 随机 h)+ dose-response——才是缩小与元目标差距的有效方向。架构缺口已列([mental_world.md](mental_world.md) §6 / §10:逆动力学头对 context 全盲、单 h 太细、缺"用 context"的梯度压力)。
- **已到上限的角落**:**onset 阈值微调**。诊断结案、收益递减,继续打磨不能缩小与元目标的任何差距。
- **判断建议**:今后衡量"是否在朝目标走",看 **in-context 适应增益**(dose-response 是否单调、是否对训练未见的映射成立),而非 onset / pred_move 的小数点。

---

## 5. 是否值得继续?

- 作为"从零造动物式智能"的**直接**实现——不是,也没打算是(§2)。以此为唯一标准会持续失望。
- 作为"**给定先天感知,研究 in-context 动力学发现与重绑定**"的命题——有真实、可证伪的进展(§3 三条已有支持),且 §4 指明了唯一仍在缩小差距的方向。
- **继续投入的前提**:重点放在 §4 的 in-context 轴,并立一个可证伪的里程碑(如"对训练**未见**的控制映射,dose-response 适应增益单调且显著")。若该里程碑反复无法达到,**那才是**重新评估整条路线的信号——而不是因为 onset 顶不过 0.5 就怀疑全局。
