# Step 3:指令总线(慢脑离散思考 → 快脑执行)

**上游**: [[fovea-twotower-step2]](S4 一档全 FAIL:离散内容经状态=溶解且有害)、
两系统讨论(2026-07-05:平滑规律进状态、离散内容走 token、检测反射不过脑)。
**用户命题**: 直接喂文本指令 token("向左转"/"瞄准敌人"),观察快脑是否真的执行。

## 1. 通道设计(v0)

指令 = 1 个钉住的 token,放在动作塔序列头([cmd | 81 vis | H 噪声动作]),
注意力可随机访问,不经递归状态(Step2 已证状态通道对离散内容失效)。
文本短语 → MiniLM(en, 384 维)固定嵌入;左右短语余弦 0.886,差异留给 cmd_in 线性层放大。

**指令来源 = 事后标注**(hindsight,自监督,无需人工):未来 0.8s chunk 里
真实发生的行为反推指令。词表 v0(en 训练 / zh 对照):
turn left/向左转(dx和<−0.5)、turn right/向右转(>+0.5)、open fire/开火(attack 任意帧)、
move forward/前进(W≥半程)、jump/跳(space 任意帧)、act freely/自由行动(空指令)。
训练时 70% 从适用集均匀采样、30% 强制空指令(保持自由行为校准)。
**"瞄准敌人"登记为二期**:需敌人检测标注(YOLO 出方位),判据=预测 dx 是否把准星推向敌人方位。

## 2. 待证命题(S6,判据先于结果登记)

| 命题 | 度量 | 过 | 败处置 |
|---|---|---|---|
| S6a 通道读取 | 转向窗 dx 方向 AUC:真指令 vs 空指令,配对 Δ | CI>0(指令泄露未来,只证 token 被读,数值不作卖点) | token 被无视 → 查 cmd_in/容量,通道实现问题 |
| S6b 反事实服从(核心) | 实际右转窗喂 turn left(+镜像):服从率 = sign(预测dx和)==指令方向;辅:可导性 Δ=P(预测右\|RIGHT)−P(预测右\|LEFT) 全窗 | 服从率 bootstrap CI 下界 >0.5 | 指令只作先验不作因果 → 快脑"参考建议"而非"执行命令",指令总线降级为提示通道,需加服从性训练(如指令加权损失) |
| S6c 特异性 | 转向指令不应移动开火预测(attack AUC 差,报告);开火可导性:无开火窗 mean pred_attack (FIRE)−(FREE) | fire_steer CI>0 且转向→开火串扰小 | 串扰大 → 指令表征纠缠,考虑词表正交化或逐槽位 token |

**S6b 过 = 用户命题成立**:慢脑的离散思考可以编译成文本 token 直接指挥快脑,
且是因果通道(可反事实操纵),为 LLM 语义慢通道接入建立机制先例。
注意与 S4b 的对照:同样的信息,走状态=有害干扰,走钉住 token=?——这一对比
本身就是"什么记忆走什么通道"分界线的第三块实证。

## 3. 训练与评测

- C1:`train/fovea_twotower/train_cmd.py`,ActionTower(n_cmd=384),W1 状态播种
  (真实消息,与 M1 一致),6000 步 bs8,指令采样独立 RNG(seed 555);
- 终判:`train/fovea_twotower/eval_cmd.py`,holdout 每窗 5 条件配对
  (FREE/TRUE/LEFT/RIGHT/FIRE,逐批同初始噪声),n=2000 窗 → runs/ftt_s6_360.json;
- 排程:W1b 链(S4a 二档)之后自动,C1 ≈35min + 终判 ≈25min(5 条件×2000 窗)。

已知限度:BC 离线口径,"服从"指预测分布被指令因果移动,非闭环执行;
闭环执行(真游戏里发指令看转不转)属 Step 1.5 闭环基建之后的事。

## 4. S6 判定结果(2026-07-05,C1 6000步,runs/ftt_s6_360.json,n=2000 窗/1022 转向窗)

| 命题 | 结果 | 数据 |
|---|---|---|
| S6a 通道读取 | **PASS** | 转向窗 dx 方向 AUC:真指令 0.989 vs 空指令 0.576,配对 Δ CI [+0.378,+0.446]>0 |
| S6b 反事实服从(核心) | **PASS** | 反事实注入服从率 **0.940**,CI [0.927,0.955]——下界 0.927 ≫ 0.5;可导性 Δ=0.913 CI [0.900,0.924];空指令随机基线 0.564 |
| S6c 特异性 | **PASS** | fire_steer 0.767 CI [0.753,0.783]>0;串扰低:开火注意力 AUC 在 LEFT 指令下 0.617 vs 空 0.626(几乎不动)|

判读(核心命题成立):把实际右转窗强行喂 `turn left`(及镜像),快脑预测的
位移方向 **94%** 翻到指令侧——**慢脑的离散思考编译成文本 token 可以因果指挥快脑**,
不是"参考建议"而是"执行命令"(反事实可操纵 = 因果通道,非仅相关)。空指令基线
0.564(近随机)证明效应来自指令本身而非窗口偏置。fire_steer 过 + 转向指令不扰动
开火注意力 → 指令表征按语义槽位解耦,未纠缠。

**与 S4b 的对照(分界线第三块实证)**:同一"该往哪走"的信息——
走 Mamba **状态**(S4b:M1 事件窗 dx AUC 0.568 < M0 0.588,显著**有害干扰**)
vs 走**钉住的指令 token**(S6b:94% 服从)。同信息、两通道、两极结果,
直接坐实"什么记忆走什么通道"的分界线:平滑规律/场景内容进状态会互相踩踏,
离散的、需随机访问的符号(指令/事件)必须走 token 通道。这为 Step4 的
LLM 语义慢通道(能力3:冻结预训练塔读消息历史→出文本指令→快脑执行)
建立了机制先例——通道是通的,且是因果的。
