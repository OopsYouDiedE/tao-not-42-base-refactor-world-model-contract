# Step 5:轨迹价值判断 → 过程优势微调(v2,2026-07-05 用户第二次细化)

**上游**: [[fovea-twotower-step3]](S6 全 PASS)、[[fovea-twotower-step4]](S7a 进行中)。
**版本注**: v1(同日早先)只登记了结果层成对比较;v2 按用户细化重构。
**v2 在任何 S8 结果产出之前登记,预注册纪律未破。**

**用户命题(2026-07-05,二次细化)**:
1. "轨迹相对好坏可行性"要拆成两问:(一)模型**有没有能力**预测;
   (二)是否**必须同一出发点**比较(若必须,数据集准备失效);
2. 预训练与 RL 微调本质不同:预训练=给定图像+动作,训的是**反射**;
   RL 微调=给定不完整观测,在模拟器**或脑内推演**,学的是**哪条长路径更有效**;
3. 关注**过程上的优势**(而非只看结果);微调后希望模型展现
   "无法直接确定成果时,进行多种尝试排除错误答案"的行为(类思维链);
4. 少量标注验证:届时开 **Sonnet agent** 做两件事——数据集是否有效、
   相对优势标注是否正确。

## 0. 核心分析(答案先记于此,判据见 §2)

### 0.1 两问正交,分开测
"能力"(冻结塔表征里有没有质量信号)与"口径"(什么配对协议下信号可读)
是两个独立轴。能力用最干净的同起点配对测上界(S8a);口径单独测(S8b)。
能力败 ≠ 口径败,反之亦然;混在一起测则任一败都无法归因。

### 0.2 同起点问题的三重消解(问题二的答案:数据集准备**不失效**)
1. **过程层消解**:过程优势的标注 = 从同一中间状态 s_t 分叉 k 条延续,
   比较 V̂(分叉后) − V̂(分叉前)——**比较天然同起点**,跨起点难题在过程层根本不出现。
   但见 §0.6:分叉标注有冷启动洞,它不能独自成立。
2. **配对消解**:结果层若确需可比性,用**起点相似度配对**(DINOv2 潜空间近邻
   匹配,倾向得分配对的思路)代替严格同 seed——数据集无需成对采集。
3. **口径消解**:regret/优势口径(CPL, Hejna 2023)理论上跨起点可比;S8b 实测
   它能否穿透"起点运气"。三条防线,任一成立数据集都不失效。

### 0.6 跨起点为什么是**存亡条件**而非辅助(2026-07-05 用户三次细化)
**用户理由**:要在模型**尚不具备控制能力时**,用数据集告诉它"怎么做是好的",
否则模型永远不会去尝试那些行为。这暴露 v2 的一个洞:
- **分叉标注的冷启动洞**:S8c 要求 rollout 策略有非零成功率——反射策略做不成
  的事,k 条延续全败,V̂ 处处≈0,过程优势恒零,**信号真空**;困难行为恰恰是
  策略支撑集之外的,自采样永远采不到。
- **数据集为师的唯一通道 = 跨起点比较**:现成数据集(g500/别人的轨迹)里
  **不存在**同起点配对;若跨起点不可比,数据集对价值学习就是废的,模型只能
  从自己的尝试学——而尝试不会发生。**S8b 因此从"口径位"升格为
  "数据集能否当老师"的存亡命题**。
- **由此管线分两相**:Phase 0(冷启动,数据集为师):跨起点 regret 分数 →
  质量加权 BC / 质量条件化 BC(AWR / Decision-Transformer 式"告诉它什么是
  好的再要求好的"),把好行为**注入策略支撑集**;
  Phase 1(有能力后,自我精修):切到 §0.1–0.4 的分叉 MC 过程优势,做细粒度
  信用分配与"多尝试排除"。**Phase 0 是 Phase 1 的前置**,预登记门槛:所选
  任务上反射策略成功率 < 5% 时,Phase 0 强制先行(否则 S8c 无意义)。
  这与 RLHF 的 SFT→RM→RL 分相同构:先把行为放进支撑集,再精修。

### 0.3 预训练(反射)与 RL 微调(推演)的接口 = 级联,不冲突
- **反射策略**(C1/W4 模仿预训练)= MC 推演的 **rollout 发生器** + 微调初始化;
- **世界模型**(W4 下帧预测)= 脑内推演器(二期;一期先用模拟器,因 AR 多步
  rollout 保真度尚未验证,见 §4 依赖);
- **过程优势** = 两者之间的信用分配信号:预训练产出推演器,推演产出优势标注,
  优势微调反射。预训练不是被 RL 替换,而是 RL 的**发生器**——这本身是
  愿景("快脑反射 + 慢脑推演"分工)的又一实证形态。

### 0.4 "多尝试排除"如何变得可信用分配(关键设计约束)
物理进度口径会惩罚失败尝试(挖错方向=负进度),把探索行为压掉。
要让"排除错误答案"得正分,优势必须定义在**信念状态**上:失败尝试若排除了
假设,信念价值上升。**实现上不需额外机器**:只要 MC rollout 策略**以历史为
条件**(我们的塔正是——GDN 状态播种/消息历史),从"失败后状态"(历史含失败
记录)起的 V̂ 自然高于"未尝试状态"的 V̂,因为后续策略已知哪扇门是错的。
即:**历史条件策略的 MC 价值自动是信念价值**。
前提:任务须有可排除的隐藏结构(部分可观测搜索类,如"找到并接近矿/结构",
错方向排除后剩余方向命中率上升)——任务选型据此定。

### 0.5 文献坐标(命题不是新的,是优势)
结果层 = Christiano 2017 轨迹偏好 + Bradley-Terry(RLHF 原始形态;DPO 为其
文本特例);过程层 = PRM vs ORM(Lightman 2023 Let's Verify Step by Step),
自动过程标注 = Math-Shepherd 式 MC 分叉;"多尝试排除"= o1/R1 式搜索行为,
但我们走过程监督使其可控涌现。模型代替人打偏好 = RLAIF。

## 1. 管线设计

**分叉标注(S8c 核心)**:采样中间状态 s_t → 以反射策略(历史条件)开 k=8 条
延续,视界 H 内成就/事件计分 → V̂(s_t)=均分;段优势 A(t→t+Δ)=V̂(s_{t+Δ})−V̂(s_t)。
分叉实现三选一(实现时登记):(a) CraftGround 存档/快照回放;
(b) 同 seed+同动作前缀重放(登记风险:生物/随机事件发散);
(c) 世界模型脑内 rollout(二期,依赖 AR 保真度)。

**Sonnet agent 两职(用户指定,届时开 Agent 执行)**:
1. **数据集有效性审计**:g500 窗口质量信号方差、事件计分分布、配对可分性
   → 判"g500 能否支撑 S8a,还是必须转 CraftGround 采集";
2. **过程优势标注抽查**:抽 N=100 高置信标注对,Sonnet 看帧序列+消息流独立
   判优,与自动标注一致率 ≥80%(高置信对上)判标注管线可信。
   判据在抽查前固定,抽查样本随机 seed 登记。
