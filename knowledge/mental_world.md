## 1. 一句话立场

把系统从"看视频→反应"改造成**脑内世界**：在内部维护一组**有限的持久潜向量**，它本身是一个可以自主运转的生成模型；
**只在偶尔观测时注入预测误差**去修正它。框架选 **Transformer**，**弃用 Mamba**。

弃用 Mamba 的理由：当初引入 Mamba 的唯一动机是"逐像素 SSM 建立不了运动对应"，其前提是"世界状态 = 像素网格特征图"。
一旦核心状态改为**有限抽象潜向量**（N 很小），该前提不再成立，Transformer 的全局注意力在小 token 集上更自然，
Mamba 的长序列线性扫描优势也用不上。

---

## 2. 核心表示：有限持久潜向量

- 世界状态 `Z ∈ [B, N, d]`：N 个**跨时间持续存在的潜向量**，不是稠密像素场。以最少数量满足任务目标。
- **残差/渐进分解（非物体中心 slot）**：slot1 承载主成分、slot2 补 slot1 未覆盖的部分、slot3 再做进一步校正……每个潜向量装"前面未装下的"。由此带来四条约束：
  - 天然有**幅度谱**（slot1 大 → slotN 小）；slot 间是**顺序正交（Gram-Schmidt 式）**，不是两两对称互斥；
  - **无实体身份可匹配**——Sinkhorn/匈牙利那套 slot 分派**不适用**（那是物体中心 slot 的做法）；
  - 重要性排序由 **nested dropout / Ordered Autoencoder** 思想诱导（随机只留前 k 个 slot 仍要求可重建），不手写正交损失；
  - **SIGReg 接线注意**：勿把 `Z` 摊成 `[1, B*N, d]` 做池化 SIGReg——大小尺度混合在 SIGReg 眼里是**非高斯（峰态异常）**，会对**幅度谱施压使其趋平**。须**方向/幅度拆开**：对方向（单位化后）防坍缩，让幅度自由成谱。
- 残余的"多个 slot 冗余绑同一区域"由**结构 + 损失**两层处理：结构上沿 **slot 维**做零和竞争（一个 patch 的注意力质量被各槽分摊），损失上软惩罚竞争注意力图的成对重叠。约束加在**注意力空间（谁看哪里）**，前向注意力仍是合法分布；**不**在前向里硬正交化（硬改聚合权重会破坏 `out=w·v` 的加权平均语义，并按 slot 序让靠后的槽缺少输入）。
- "每次更新完全保持潜向量"= **结构上**潜向量身份/数量不变（持久）+ **算子上**每步只做受限增益的残差改写（非扩张，I5）。
- 范式：**预测编码 / 主动推断**，落成 **predict–correct 两相**（类 Kalman / RSSM）。

## 3. 与 LeWM 的区别（本设计的定位）

本设计**借用** LeWM（github.com/lucas-maes/le-wm）的内核，但在六处有意偏离。"这不就是 LeWM 吗"的疑问，大概率对应的是共享内核部分——对的。

**共享内核（= LeWM，照搬，不是本设计的创新点）**：JEPA 潜空间预测（不解码回像素）+ SIGReg 防坍缩 + Transformer + 全观测提供稠密 target。

**六条真区别（本设计的选择）**，并标注状态（已落地 = 退役活模型曾实现；愿景 = 无对应代码；方向 = 设计目标）：

| # | 维度 | LeWM | 本设计 | 状态 |
|---|---|---|---|---|
| 1 | **读取** | 每帧全读、全帧全 ViT | **选择性读**：中央凹（空间稀疏）+ 偶尔睁眼（时间稀疏），靠蒸馏逼近全读 | 愿景（§8） |
| 2 | **状态** | 每帧一 token 的滑动窗 | **N 个持久残差潜向量**（渐进分解，非物体 slot），就地改写（RSSM 式信念） | 设计锚点 |
| 3 | **索引** | 严格帧序号、规则稠密 | **(时间, 空间) 标签**，稀疏不规则事件 | 部分（时间：可变 Δt + 连续时间编码；空间标签属愿景） |
| 4 | **输出** | 只出下一帧嵌入 | 世界信念 Z（探针可读）+ **动作潜向量→定时动作序列** + gaze/wake | 部分（gaze/wake 属愿景） |
| 5 | **可控性** | 隐式动作条件 | **显式可控闸 c**（逆动力学接地）+ **随机隐变量 ξ**（吸收不可控分量） | 设计锚点 |
| 6 | **用途** | 给规划用的世界模型（试动作 MPC） | 快速迁移底座 + 实时指导 + 推理期反应式策略，世界模型退训练期 | 方向（§6） |

注意 SIGReg **不在**区别里——它是共享的，本设计借用。区别全在"怎么读、状态长啥样、输出什么、用途"，不在防坍缩。

## 4. 两层目标：不要把元目标当成操作命题

- **元目标（meta）**：AI 像动物一样，自主从场景交互里发现因果，不依赖大规模预训练。
- **操作目标**（§6）：看一段视频 + 试跑几步，**权重冻结**下 in-context 适应新玩法。

操作目标是元目标在"可证伪 + 能在现有算力上测"约束下的**收窄版本**，经两步缩减：(a) 把"发现因果"收窄为"读出 / 在线适应 action→effect"；(b) 把"不依赖预训练"这一约束暂时放掉（见 §5）。**认清这是收窄而非等价，是校准的第一步**——否则任何具体指标都会显得"离目标太远"，因为它本就只覆盖目标的一个子集。

---

## 5. 预训练立场：不打算"从零学感知"

- **事实**：感知使用冻结 DINOv2/v3（海量图像预训练），控制语义使用 MiniLM（文本预训练）。元目标字面上的"不依赖庞大预训练"，当前系统**不满足**。
- **为何这样选**：从零随机骨干无先验、holdout 退化（实测）；冻结预训练骨干给 JEPA target 一个独立于本任务数据的意义来源，是 JEPA 的前提；现有算力下也训不出可用的感知。
- **重新定位**：动物不是白板。视觉皮层是进化"预训练"好、出生即带的先天结构。把冻结骨干类比为"先天的视觉皮层"，则"冻结感知先验 + 从交互学当前环境的动力学 + 在线重绑定"比"一切从零"**更**接近动物的学习方式——动物一生学的也正是这个环境里的因果，不是重学视觉。
- **结论**：把元目标的硬约束从"零预训练"修正为——**感知先验可借（类比先天皮层），但因果 / 动力学必须从 agent 自身交互里学、且能在线适应**。在这条修正后的标准下，"用了 DINOv2"不算偏离，不必为此纠结。

## 6. 北极星：看视频掌握玩法（in-context，方向）

**当前北极星，表述为一条可证伪命题**：把"用试错学会玩法"压成"看一段视频 + 试跑几步，靠**脑内记忆**在**权重冻结**下 in-context 适应新玩法"。

- **验证杠杆**：训练/测试**环境分布 disjoint**——测试时遇到的控制映射/任务在训练里没出现过，唯一能解释适应的就是 in-context（h 记忆）而非权重已记住答案。
- **脚手架**：控制重映射（逐 episode，零新数据即制造"同一画面、不同操作语义"的 in-context 信号）；任务条件通路（原 task_text.py）已随退役管线删除，待重建。
- **证据档**：① 逐 episode 操作重映射（看一段被重映射的视频 → 推出本 episode 的操作语义）；② episodic loss-on-query（在 query 段上算适应增益）。
- **评估协议**：baselines（无记忆 / 随机 h）对照 + dose-response（适应增益随 context 视频长度单调）。真环境（headless 软渲染）解锁主动交互轴 + rollout 通关率，离线数据测不了这些。
- **已知架构缺口**（待补，非阻塞）：逆动力学头对 context 全盲（看不到适应信号）；单个 h token 容量有限，承载不下一整套玩法记忆；缺"有用 context"的梯度压力（模型没有动机真去用 context）。这些是把方向变成结果前需要解决的问题。

## 7. 诚实边界：哪条线朝元目标走

- **判断标准**：衡量"是否在朝元目标走"，看 **in-context 适应增益**——对训练**未见**的控制映射，dose-response 是否单调且显著——而非任何单一读出指标的小数点。Δz-JEPA 线在此前曾连续数轮在 onset 单指标上打磨，那是从大目标滑向代理任务的子角落的典型，已诊断结案、收益递减。
- **朝元目标的有效方向**：in-context 因果发现 / 重绑定（§6）。把它做扎实——更多键、训练没见过的重映射 / 效果、拉长适应曲线、加 baselines（无记忆 / 随机 h）+ dose-response——才是缩小与元目标差距的方向。
- **是否值得继续**：
  - 作为"从零造动物式智能"的**直接**实现——不是，也没打算是（§5）。以此为唯一标准会持续失望。
  - 作为"**给定先天感知，研究 in-context 动力学发现与重绑定**"的命题——这是有真实、可证伪进展空间的方向。
  - **继续投入的前提**：立一个可证伪的里程碑（如"对训练未见的控制映射，dose-response 适应增益单调且显著"）。若该里程碑反复无法达到，**那才是**重新评估整条路线的信号。

## 8. 更远的愿景：选择性读取（fovea / gaze / wake，无代码）

本节描述**尚未落地**的设计愿景，**当前仓库无对应代码**。保留它是因为：当"读取成本"成为瓶颈时，这是既定演进方向。

- **双通道感知 = 余光 + 中央凹**：余光廉价常开、给全局显著性；中央凹只对注视点的局部裁剪做高清 ViT。这是"偶尔扫一眼"的实现方式——不把整帧高清灌进模型。
- **主动感知出口 = gaze**：决定下一眼看哪 `(g_x, g_y, g_s)`，经可导的 STN（affine_grid + grid_sample，fp32 满足 I4）裁剪——**"看哪里"可被梯度直接学**，不必退回高方差硬注意力 RL。空间标签用 `SpatialPosEmbed`（已在 [blocks/encodings.py](../blocks/encodings.py)，待接入）。
- **异步节律 = wake**：输出"多久后再睁眼"，配合连续时间编码处理稀疏/不规则观测。
- **训练靠两阶段自监督蒸馏（绕开 RL）**：阶段一全观测训出动力学与表征；阶段二冻成老师，gaze/wake 用**从自身预测误差算出的 oracle 当自监督标签**。**观测必须有代价** `cost(obs)`——否则"最大降低预测误差"的最优解是全看，退回稠密感知；加入代价才能逼出稀疏读取。

## 9. 证伪探针与诊断原则

- **薄探针原则**：检验"Z 是否真存了世界"的探针须**刻意保持简单**（单隐层），且从**聚合后的全体残差潜向量**读出、非某个固定 slot——读得准只能归功于 Z 本身确实编码了世界信息，而非探针自行拟合。
- **指标约束**：① 低 loss 不等于世界模型有效，须用抗作弊指标（开环对照，骗不了位置先验）；② 多实体读出用结构化 query、不池化；③ 运动样本要单独看（全步 RMS 会被静止帧稀释）。
