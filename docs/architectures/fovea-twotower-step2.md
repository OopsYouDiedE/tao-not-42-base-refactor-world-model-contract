# Step 2:慢通道指挥下的快通道盲区补偿 × Mamba 带时记忆

**上游**: [[fovea-twotower-design]]、[[fovea-twotower-step1]](终审:播种机制成立,
世界记忆增量真空)、两系统讨论(2026-07-05:语义=长程规划,战术记忆=两不管地带)
**核心假设(用户命题)**: 快通道(凹区)能在慢通道指挥下弥补固有缺陷;慢通道传入的
信息可做时间编码,在 Mamba 状态中形成**带时间的记忆**。成立则为凹视双传入结构铺路。

## 0. 实验母题与"慢通道"的替身

真慢通道 = 27B 语义/战略流,本实验用**周边运动事件消息**替身:同为"稀疏、带时间戳、
凹区看不见的外部信息注入",管道机制(注入→Mamba 保持→策略消费)完全同构,
只有内容和频率不同。替身的额外好处:它就是设计 §2.1 的**变化通道**,
本实验同时是凹视双传入(凹区+周边)的 v0。

**盲区结构天然存在**: 360p 分片存完整 640×360 帧;快通道=center 126 原生裁剪
(35%高×20%宽),周边 ~93% 面积凹区不可见;而人类玩家看的是全屏——
其动作里含周边信息,快通道模型只能靠注入的消息弥补。

## 1. 消息通道(变化通道 v0,dataloader 在线提取,零预处理)

每帧 1 个消息 token(11 维):
- 全帧灰度缩至 80×45,帧差 3×3 平滑;掩膜=视口中环(行8:37,列10:70)去中心 16×16
  ——**必须排除边缘 HUD 带**(小地图/击杀播报/弹药永动,校准时单像素 max 中位数达
  132/255,事件率虚高到 0.74);
- bg=周边中值(相机自运动代理),peak/位置=周边最大值;
- **事件** ⇔ peak−bg>45 且 bg<3(2026-07-05 于 360p holdout 校准,事件率 8.6%);
- 特征 = [8 方位 one-hot(事件门控), log 显著度, log 背景(pan 代理), valid]。

时间编码:消息 token 按真实帧序进流(与视觉/动作交错),"带时间的记忆"
= Mamba 状态对 (事件内容 × 事件距今时长) 的联合保持——由探针直接读出验证。

## 2. 待证命题(可证伪)

- **S4a 记忆形成**: 冻结 W1 塔,线性探针从 **STATE**(9层recurrent_state,step1 教训)
  读出"最近周边事件的方位"(事件在 [t−3s, t−0.2s],当前无事件——强制走记忆)
  且**回归事件距今时长**。判据:方位 acc ≥ FRAME 探针 +5pt 且 时长回归 R² CI>0。
- **S4b 缺陷弥补(核心)**: 动作塔只经播种读历史(消息不直通策略),
  在"周边事件后 1s 内"窗口上,带消息组 显著优于 消息置零组。
  判据:dx 转向方向 AUC 配对 bootstrap CI>0(主),attack AUC(次)。
- **S4c 时间编码被使用**: 消息**时序打乱**组(内容边际保留、时间对齐破坏)
  显著劣于正常组。判据:S4b 主指标 M1−Mscr CI>0。
  不过 = 记忆是"事件袋"而非带时记忆,时间编码主张不成立。

## 3. 训练与消融矩阵(全部同一 --rng 配对)

| 编号 | 配置 | 验证 |
|---|---|---|
| W1 | Context 塔:凹区 81 tok + 消息 1 tok + 动作 1 tok /帧,下帧 MSE **不变** | 消息在 MSE 压力下被自发保持?(周边动静迟早入凹区→记住消息有预测价值,无循环) |
| PB | 探针:STATE / FRAME / MSG-HIST(裸消息缓冲,信息上界对照) | S4a |
| M1 | Action 塔:播种自 W1(真实消息流) | 主案 |
| M0 | 同 W1 ckpt,状态计算时消息**置零**(同塔同参,只删信息) | S4b 对照 |
| Mscr | 同 W1 ckpt,消息**窗口内时序打乱** | S4c 对照 |

M0/Mscr 不重训世界塔——同一冻结塔在不同输入下取状态,消融最干净。

## 4. 判据与处置

| 命题 | 过 | 败处置 |
|---|---|---|
| S4a | 方位 acc ≥ FRAME+5pt 且 age R² CI>0 | MSE 免费搭车失败 → 二档:加"预测下一消息"辅助目标重训 W1(与探针目标"回忆过去消息"不同,机制主张仍可证,但记为"需显式目标"档) |
| S4b | Δ(dx方向AUC) CI>0 于事件窗 | 信息在状态里但策略读不出 → 播种读出问题,比 S4a 败更伤设计,查 seed 接口 |
| S4c | M1−Mscr CI>0 | 时间编码未被使用,记忆=事件袋 → 时间戳主张降级,双传入仍可行但降为无时序版本 |

**S4a+S4b+S4c 全过** = 用户命题成立:慢通道→带时 Mamba 记忆→快通道盲区补偿
全链路实证,凹视双传入进入正式设计,且为语义慢通道(同管道换内容)建立机制先例。

## 6. S5:脑内世界正面度量(what's right,2026-07-05 预登记)

**动机(用户指令)**: S1–S4 全是消融式判据(去掉X掉多少),缺一个正面度量
"Mamba 状态里装的世界有多大、能否前推、可否操纵"。S5 在**冻结 W1 塔**上零训练完成
(`train/fovea_twotower/eval_s5.py`,管线结束后自动跑)。

三属性,判据先于结果登记:

| 命题 | 度量 | 过 | 败处置 |
|---|---|---|---|
| S5a 记忆视界 | 从 STATE 线性重建 t−k 帧潜变量(3×3 池化,tgt_norm 域),对比 FRAME(单帧零历史);k∈{2,5,10,20} 全曲线 | 注册点 k=10(1s 前):ΔR²(STATE−FRAME) 配对 bootstrap CI>0 | 状态只装"现在"不装"过去场景"→ 脑内世界=薄状态,与 S4a 交叉核对(消息记忆 vs 画面记忆可分离) |
| S5b 前瞻视界 | 重建 t+k 帧,**两臂同给**已执行动作序列(plan 条件公平);COPY(t 帧当预测)作参考线 | 注册点 k=+10:ΔR² CI>0 | 状态含历史但对未来无增益 → 记忆库而非前瞻模拟器;与 Step1"世界记忆增量真空"边界一致,1.5B 语义前不宜宣称"脑内世界" |
| S5c 可控性 | 同一前缀,末动作 dx 真实 vs 翻符号,读 act 位对下帧首 patch 的预测误差(限 \|f(dx)\|≥0.5 锚点) | Δerr(flip−true) CI>0 | 预测不是动作条件化的 → efference copy 没被当输入用,"模拟器"主张不成立,只是缓存 |

三过 = "Mamba 天然充当脑内世界"从口号变实证:有过去、能前推、可操纵。
辅助输出:R² 随 k 的完整视界曲线(状态的时间视界首次被量化)。

**已知限度(诚实声明)**: S5b 的"前推"是线性探针口径,不是自回归盲滚动
(逐 token AR 需给 CausalAttn 加 KV cache 增量解码,登记为 Step3 候选);
S5c 只读下帧首 patch(act 位唯一可读的免污染预测位)。

## 7. 一档(免费搭车)判定结果(2026-07-05,W1 6000步 eval_loss 0.199)

| 命题 | 结果 | 数据 |
|---|---|---|
| S4a | **FAIL** | 方位 STATE 27.1% vs FRAME 24.0%(Δ+3.1pt<5pt,机会率 12.5%);时长 R² CI [−0.19,+0.02] 跨零;**MSG-HIST 上界 77.1%/0.68 → 信息充足,塔不记** |
| S4b | **FAIL(显著反向)** | 事件窗 dx 方向 AUC:M1 0.568 < M0 0.588,Δ CI [−0.037,−0.003] 全负;无事件窗 Δ 零(+0.004)→ 损害特异于事件窗 |
| S4c | FAIL(零) | Δ(M1−Mscr) CI [−0.019,+0.005] 跨零 |

判读(链式一致,非三个独立失败):记忆没形成(S4a)→ 补偿无从谈起(S4b)→
时间编码无从使用(S4c)。病根与 Step1"世界记忆增量真空"同源:**纯 MSE 下帧预测
不奖励保持稀疏事件**。S4b 显著为负的解释:消息 token 在事件窗扰动了状态,
但内容策略读不出 → 纯干扰项。两点诚实保留:(1) 每臂 n=1 训练,逐窗配对 CI
不含跑间方差(RNG 配对缓解但梯度轨迹会发散放大),负号解释降权;
(2) 非注册指标 keys 宏 AUC M1 0.661 > M0 0.630(+3.1pt),
消息对整体按键预测可能有正贡献,记为探索性信号。

**处置(§4 预登记)**: 走二档 W1b(+消息位预测下一帧消息,权重 1.0)已排队;
若 W1b 使 S4a 过档,则命题降级为"带时记忆可形成,但需显式目标,免费搭车不够",
届时用 W1b 重训 M 臂复判 S4b/S4c(另一夜)。

### 7.1 二档 W1b 判定(2026-07-05,aux-msg 1.0,6000步 eval_loss 0.218)

| 命题 | 结果 | 数据 |
|---|---|---|
| S4a 复判 | **PASS** | 方位 STATE 29.6% vs FRAME 24.4%(Δ+5.2pt ≥5pt);时长 R²:STATE **+0.275** CI [+0.206,+0.343] vs FRAME −0.14(CI 全负);MSG-HIST 上界 77.4%/0.69 未变 |

判读:加"预测下一帧消息"辅助目标后,同一架构、同一数据、同一探针,Mamba 状态
从"完全不记"(一档 Δ+3.1pt、age R² 跨零)变为**同时保持事件方位与距今时长**

分界线论:纯 MSE 只学平滑规律;要状态装离散事件,目标函数得点名要它。

#### 7.1.1 W1b 上的 S5 对照(探索性,无预登记门;runs/ftt_s5b_360.json)

问题:aux_msg 修好了带时记忆探针(S4a),是否**顺带**改变场景记忆/前瞻视界?
方法(预登记):同塔跑 S5a/b/c,对比 §8 的 W1 数字,只描述不设门。

| 指标 | W1(§8) STATE−FRAME ΔR² CI | W1b 同 CI | 走向 |
|---|---|---|---|
| 记忆视界 past2 | [−0.022,−0.0005] | [−0.013,+0.008] | 微升,跨零 |
| 记忆视界 past10 | [−0.262,−0.234] | [−0.301,−0.274] | **更负** |
| 记忆视界 past20 | [−0.420,−0.390] | [−0.501,−0.469] | **更负** |
| 前瞻视界 fut10 | [−0.440,−0.403] | [−0.445,−0.410] | 几乎不变 |
| 可控性 S5c Δerr | +0.0045,CI [+0.0014,+0.0081](PASS) | +0.0033,CI [−0.0003,+0.0065](**跨零**) | 减弱 |

读法(分界线口径):这是一次**干净的解离**。aux_msg 抬高了带时探针
(S4a 方位 +5.2pt、age R² +0.275),却**没有**改善情景性场景召回(S5a:短滞后微升、
长滞后反而更负,全程深负)与前瞻(S5b 基本不动)。正是程序性/情景性分界所预期:
显式目标让状态装**程序性时间簿记**(发生了什么、距今多久、哪个方位),
但不把状态变成**情景性场景存储**——目标指向离散事件 token,不指向 patch 重建。
状态仍非帧缓存,与 S4a/S5a 交叉一致。

一处诚实的皱褶:最强的单项正面信号——一步可控性 S5c——在 W1b 权重上 CI **跨零**了
(点估计 0.0045→0.0033,降 ~26%)。这**不撤销** W1 的 PASS(不同权重),但坐实
S5c 在 n=1 下**脆弱**,亟需 2–3 种子复现(已在债务清单)。当前证据强度:S5c 应记为
"单次 PASS,待复现",而非稳态结论。

## 8. S5 判定结果(2026-07-05,冻结 W1 塔,runs/ftt_s5_360.json)

| 命题 | 结果 | 数据 |
|---|---|---|

判读(程序性/情景性分界口径,2026-07-05 用户重构后 S5a/b 低分 = 确认而非意外):
状态通道**不是情景性场景存储**(与 S4a 交叉一致);但 **S5c 过 = 首个正面"脑内世界"性质**
——下帧预测是动作条件化的,efference copy 被当输入用,模型是(至少一步深的)
**模拟器而非缓存**。这是分界线两侧各得其所的又一实证:动作→后果 = 平滑规律,进状态/权重;
场景内容 = 情景,状态不装。诚实保留:STATE 臂 R² 深度为负提示池化读出 + 固定 α 岭回归
可能低估状态含量(读出瓶颈),但方法预登记,判定不改;S7b(Step4 预训练塔复测)
将区分"状态天生薄" vs "从零小塔的状态薄"。
辅助发现:COPY 参考线 k=10 即 −0.27——场景 1s 内强去相关,长视界重建对任何臂都是硬任务。
