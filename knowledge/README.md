# 项目知识库

> 本目录只收录**外部事实**:论文、官方文档、规范、指南、建议——凡涉及事实/规范/指南/建议,优先引用权威官方原文,**禁止以模型自身知识库作为论据**。项目私有决策(方向、约定、退役教训)不在此,记在会话记忆库。代码是逻辑事实来源,可从代码直接读出的内容不在此重复。

## 更新规范

1. 涉及事实/规范/指南/建议时,优先用权威官方内容;禁止拿模型自身知识当论据。
2. 因**拒绝访问**(而非互联网不存在)查不到的,必须标注「ACCESS DENIED」,区别于「不存在」,不得假装某事不存在。
3. 涉及较新内容,**不信任模型知识库判断,优先上网查证**。
4. 企业公布自家产品数据等**利益相关**不降信源字母,条目标注「利益相关」并从严交叉证实。

### 信源分类(只看历史记录,不看知名度)

| 级 | 标准 | 举例 |
|---|---|---|
| A | 该话题下公认权威专业、无利益干扰、历史可靠 | 顶级期刊、国家权威指南、正统行业规范 |
| B | 权威/专业但该话题偶有轻微问题 | 国际主流媒体、主流期刊、机构调查报告 |
| C | 权威性存疑但过去有有效信息记录 | 媒体评论、专业人士建议、名人发言 |
| D | 有重大疑问、记录时好时坏 | 一般媒体评论、社交发帖 |
| E | 有编造/无效信息史,或无法溯源/已被证伪 | 不可信官媒、造假机构 |
| F | 无往绩、无从判断(≠不可信,全新来源默认F) | 首次接触的机构/个体 |

### 内容可信度

| 级 | 标准 |
|---|---|
| 1 | 确凿原始出处 + 多个独立可信来源交叉证实 + 逻辑自洽 |
| 2 | 确凿原始出处 + 逻辑自洽 + 与已知一致,但缺独立证实 |
| 3 | 原始出处无法考证,但多来源交叉印证 |
| 4 | 仅单一可信信源,或暂无可比信息 |
| 5 | 被更可信信息反驳/已证伪/与常识不符且无凭据 |

无法判断的全新内容按 5 保守处理但标注「无从判断」。时效性不参与 1–5,记在时效栏。

### 使用门槛

可信度 1–2 可进正文;3–4 可进正文但须注明局限;5 及 E、F 类信源不得作正文论据,只可标注后作背景参照(F 补齐记录后重评)。交叉证实要求来源相互独立,同一上游转述只算一个来源。

### 每条格式

```markdown
搜索时间 | 搜索内容 | 评级(信源字母+内容数字) | 时效(更新于/失效条件)
---
知识内容
---
来源列表和原文摘引
```
