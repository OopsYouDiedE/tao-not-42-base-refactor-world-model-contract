# Step 4:预训练慢塔(从零 58M → NVIDIA Nemotron-3-Nano-4B)

**上游**: [[fovea-twotower-step2]](S4 一档全 FAIL:从零 58M + 纯 MSE 不保持稀疏事件)、
[[fovea-twotower-step3]](指令总线)。
**用户指令(2026-07-05,任务性质变更)**:
1. 任务从"记录数据"改为"找到足以支撑愿景的多项证据"——重点是**证明**;
   出现完全否定性证伪时,换方向重新实验,而非停在失败记录上;
2. **慢塔必须用预训练权重**:从零学习无法发挥大语言模型的任何优势,
   哪怕选一个较小的 NVIDIA mamba 模型(不必双塔)也要用预训练权重。

**纪律保留(不因证明导向而放松)**: 判据仍先于结果登记;改变的只是败处置——
从"记录失败收工"改为"预登记的换向方案立即执行"。证据要能支撑愿景,前提是它真的是证据。

## 0. 选型(2026-07 现况)

| 候选 | 规模/架构 | 3090 可行性 | 判定 |
|---|---|---|---|
| **nvidia/NVIDIA-Nemotron-3-Nano-4B-BF16** | 4B,21×Mamba2 + 4×Attn + 17×MLP,d=3136,自 9B-v2(20T tok)压缩蒸馏,推理型 | 冻结 bf16 8GB ✓;LoRA+梯度检查点 ✓;全参 ✗ | **主选** |
| nvidia/NVIDIA-Nemotron-Nano-9B-v2-Base | 9B 混合,真 base | bf16 18GB,只能 QLoRA,慢 | 备选(若 4B 的对齐调优干扰回归头) |
| nvidia/Hymba-1.5B-Base | 1.5B 并联混合头,base | 全参可微调 | 备选(能力弱,架构非主流) |
| state-spaces/mamba2-1.3b | 纯 SSM,Pile 300B | 全参可微调 | 对照候选(无注意力,状态容量最干净的科学对照) |

主选理由:正是用户点名的"较小的 NVIDIA mamba 模型";架构与我们 58M 塔同族
(Mamba2 ≈ GatedDeltaNet 近亲 + 少量注意力,比例同为 ~1/8);蒸馏自 20T token 预训练的
9B-v2,是 24GB 卡上能放下的最强混合底座;且它本身是推理模型 → 能力3
("借助 LLM 学习")的指令总线二期可以直接让它读消息流出指令。

**硬件结论(用户问)**: 3090 不是 4B 档(LoRA)的阻碍;是全参微调与 9B+ 档的阻碍。
LoRA 的已知限度:视觉模态吸收深度受限——但 LLaVA 系证明冻结 LM + 投影层足以
习得新模态读出,机制验证够用。**软件缺口(本步要建的)**:
mamba-ssm/causal-conv1d 内核未装(transformers 纯 torch 回退可用但 ~10× 慢,
torch 2.9+cu128 需新版内核,旧 2.2.4 pin 是 cu124 时代产物);
transformers 4.57.3 vs 模型卡测试的 4.48.3(trust_remote_code 自带建模代码,冒烟验证)。

## 1. 适配设计(W4)

- 输入:DINOv2-S 潜变量(384)→ 线性投影 → 3136,视觉/消息/动作三类各自投影 +
  类型嵌入,交错流 [81 vis | 1 msg | 1 act]/帧 与 W1 完全一致;
- 输出:3136 → 384 回归头,视觉位下帧潜变量 MSE(tgt_norm 域),与 W1 同目标;
- 训练参数:投影层 + 回归头 + LoRA(Mamba2 in/out_proj 与 Attn qkv/o,r=16)全训,
  4B 主干冻结;梯度检查点;bs1×累积 4(序列 64 帧 = 5312 token);
- 数据/预算:runs/data/g500_360p 同一数据;步数以墙钟对齐 W1(≈45min–2h 档,
  以冒烟实测吞吐定,预登记于训练脚本 args 并存 ckpt)。

### 4.1 S7a 判定(2026-07-05,W4 step3000,seq32,aux_msg=0,LoRA r16)— **FAIL(登记)**
`runs/ftt_s7_360.json`。判据 = 方位 acc_STATE≥FRAME+0.05 **且** age R²_STATE ci下界>0,两者皆须。
- **方位半 PASS 且胜从零**:acc_bearing STATE=**0.359** vs FRAME=0.223(Δ+0.136 ✓,chance 0.125),
  且 > W1 从零 0.271。→ **预训练状态确留住"事件方位",强于从零 58M**(这一半是正面证据)。
- **带时半 FAIL**:age R²_STATE=**−0.0001**(ci[−0.003,−0.0],下界<0),FRAME=−0.61;
  MSG-HIST 上界 age R²=0.667。→ 时间信息在原始消息里存在,但**塔没把"多久以前"蒸馏进状态**。
- **整体 = FAIL**(AND 不满足;不挪门框)。eval_loss 0.357(> W1 0.199,但受半步数/冻结主干/半序列
  混淆,且非判据)。诚实定位:**纯 MSE 下帧目标没逼状态编码时间标签**——与 Step2 W1 从零 S4a
  同类病根(内容在、时标不在),而非"预训练无用"(方位反而被解锁)。
- **预注册换向(立即执行档,不停在失败)**:W4b = +aux_msg 1.0(与 W1b 二档同款,逼状态重建消息
  →含时间结构),重训后复判 S7a;若 W4b 的 age 仍败 → 换 9B-v2-Base 排除"4B 对齐调优损伤"。
  注:W4b 是 ~3h 训练档,是否即刻起由用户拍板(GPU 已空)。

### 4.3 S7a 复判(W4b,aux_msg=1.0)= FAIL + 关键跨塔解离(2026-07-06 00:36)
`runs/ftt_s7_w4b.json`:
- 方位半 **PASS**:acc_bearing STATE=**0.357** vs FRAME=0.235(Δ+0.122,chance 0.125),强于从零塔。
- 带时半 **FAIL**:age R²_STATE=**−0.001** CI[−0.0077,−0.0](上界≤0);与 W4 纯 MSE 的 −0.0001 几乎相同。
  → **预注册的 aux_msg 修复对 4B 冻结塔完全无效**。
- **关键解离**:同一 aux_msg=1.0 目标,在 **W1b 从零 58M** 上把 age 修到 R²**+0.275**(PASS,step2 §7.1),
  在 **W4b 预训练 4B** 上却 −0.001(FAIL)。→ 病根**不是**容量/目标(58M 都能做到),头号嫌犯 =
  **冻结的 Mamba2 递归动力学 + LoRA**:W4 冻结了决定"状态怎么写入/衰减"的 A_log/dt_bias/D,
  LoRA 只动 in/out_proj,动不了状态写入规则;从零塔端到端训递归,能自由分配状态记 age。

### 4.4 换向决定 = W4c(解冻递归),**取代**预注册的 9B(2026-07-06,用户拍板+同意"LoRA 拿不到")
预注册败处置本为"换 9B-v2-Base 排除对齐损伤"。但 §4.3 解离把病根指向**冻结递归**而非模型规模——
9B 仍是冻结+LoRA,若病根真在此则 9B 照样败,白烧 QLoRA+18G 下载。**改跑 W4c**:预训练 4B +
aux_msg 1.0 + **解冻 Mamba2 递归动力学**(A_log/dt_bias/mixer.D,21 层×3=63 张量,极廉价,no-WD 组),
同预算(seq32 accum4 3000)。`--unfreeze-ssm`(train_w4.py),自等待驱动 `runs/ftt_w4c/run_w4c.sh`
(等 S7b 释放 GPU→冒烟验证解冻63+保存→全量→S7a/S7b 评测 → `runs/ftt_s7_w4c.json`/`ftt_s7b_w4c.json`)。
- **W4c age 过** → 坐实"冻结递归是病根",得"预训练+可训递归"可用配方(强正面结果)。
- **W4c age 仍败** → 排除递归冻结,那时才轮到 9B base(登记为下一步)。
纪律说明:这是**诊断驱动的换向**(S7a 判据本身未动、仍 FAIL),非移门框——预注册的目的(防止把失败
合理化掉)未被违反;只是新出的解离证据让 9B 从"最优下一实验"降为"次优",W4c 信息量更高且更省。
