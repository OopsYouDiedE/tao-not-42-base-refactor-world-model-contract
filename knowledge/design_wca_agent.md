# 设计：W/C/A 三模型智能体（用户 2026-07-02 拍板的架构）

> W = 世界模型（可**快速削除旧物理规律**）；C = **合理性判定**模型；A = **脑内执行动作**
> 的行动模型。硬约束（用户原话级）：**A 绝对禁止直接使用内容（原始观测/像素）作输入**，
> A 只能吃 W 的隐状态。服务主线 [[project-goal-mine-stone]]。

## 1. W：世界模型 + 快速物理置换面

- 主干：多游戏预训练的 Dreamer4（tokenizer + SpaceTimeTransformer + shortcut 头），
  吃 gaming-500-hours 通用动作-动力学对（见 design_gaming500_hd_pretrain.md，该文档已删，git 历史可查）。
- **"快速削除旧物理"的机制**：微调面 = action_proj + 各层 AdaLN 调制 Linear
  （零初始化，~4% 参数）。换游戏时只训此面（可配更高 lr），旧游戏物理以残差调制
  方式被新游戏快速覆写，主干视觉/通用动力学保留——这正是"削除旧规律"而非
  "重学一切"。已定路线：多游戏预训练 → Minecraft 小面微调。
- 验收口径：微调后 EV(Δz)、8 步开环优势 vs 从头训（同预算）。

## 2. C：合理性判定模型

- 职责：判定"W 想象出来的未来"是否合理。**核心动机是防 A 作弊**——A 在想象中
  训练时会主动搜索并利用 W 的误差区（物理漏洞）刷虚假回报；C 对不合理轨迹
  降权/截断，是想象训练的免疫系统。
- 实现候选（按代价升序）：
  1. 现成种子:难度头 D(ctx) 已存在（预测流误差）,扩展为 D(ctx, z_next) 给
     "该转移合理度"打分；
  2. 真/生成判别器：正样本=真实转移 (ctx, z₁)，负样本=W 生成的 (ctx, ẑ₁)，
     二分类头，训练几乎免费（两类样本都是训练副产品）；
  3. 集成分歧：多头/多种子 dynamics 对同一转移的预测方差,分歧大=出了数据流形。
- C 与 reward 头分工：reward 说"这好不好"，C 说"这真不真"。想象回报按 C 门控。

## 3. A：行动模型（想象中训练，禁触原始内容）

- **硬约束：A 的输入只有 W 的隐状态（token/ctx），永不接像素/原始观测。**
  推论：
  1. A 的训练与执行都经过 W 的表征——外观变化（材质包/光照/新游戏）由 W 吸收，
     A 天然可迁移；
  2. 想象训练全程隐空间推演，无解码开销，速度=隐空间前向；
  3. VPT 行为克隆先验也必须以隐状态为条件蒸馏（z→a），不是像素→a。
- 训练闭环：A 在 ctx 上出动作 → W 隐空间推演 → reward 头（成就奖励)+C 门控
  → λ-回报策略梯度（Dreamer 式 actor-critic）；CraftGround 真实数据流持续
  微调 W 的快速置换面，防想象漂移。
- 现有资产：config 已含 actor/critic 头超参；train/craftground 有
  achievement_rewards（mine_wood→mine_stone 链）与 PPO 骨架。

## 4. 自纠错回路与 C 的职责边界（用户讨论定调，2026-07-02）

用户猜想（正确）：垃圾预测扭曲的动作在**真实执行**时会走进 W 的误差区,采回的真实
转移恰好修正该误差——策略追着模型误差跑、数据反填,即 Dyna/Dreamer 的主纠错回路。
本架构与经典设计（Dreamer 想象 actor-critic）同族。

C 不替代此回路,补三个缺口：
1. **窗口期**：闭环仅在真实执行发生时闭合;纯离线/想象阶段无反馈,且想象更新比
   真实步快数个量级——A 可在一个物理漏洞上刷爆策略,纠错数据还没到。
2. **流形外**：物理不可能的幻觉状态真实执行无法复现,永远得不到真实标签;
   判别器/集成分歧直接检测出流形,无需标签。
3. **样本经济**：靠真实交互兜底 = 拿最贵资源当免疫系统;C 是零成本前置过滤。
经典对应:Dreamer 短想象步长(H≈15)限误差复利、MBPO/PETS 集成不确定性惩罚——
C 为同类对策的低成本实现。

## 5. 时序衔接

多游戏预训 W（进行中，128²/S=64 基线已启动）→ Minecraft 置换面微调 + C 判别头
（同一批数据顺带训练）→ A 想象训练（BC 先验热启动）→ CraftGround 在线闭环
→ mine_stone 验收。

## 6. 硬件预留:A 阶段的 NVFP4 量化推理(2026-07-02 讨论)

A 想象训练 = 冻结的 W 做海量前向推理(百万级 rollout、权重带宽受限)——正是
NVFP4(Blackwell 4bit 块缩放格式,16 元素/块 + FP8 缩放因子)的教科书工况:
吞吐约 FP8 两倍、显存减半。路径:租消费级 5090(推理侧生态成熟,TensorRT Model
Optimizer PTQ)而非买卡;训练侧 FP4 只有数据中心 Blackwell(B200)生态可用。
验收口径现成:量化后跑 EV(Δz)/开环优势/recon 与 bf16 对比,C 模型兜底量化噪声
导致的物理失真。当前 L4(Ada)无 FP4 运算单元,只借其存储思想——token cache
以 4bit 块缩放落盘(~3KB/帧,较 fp8 再省一半),加载解包回 bf16,需先验证
量化-反量化的 recon 掉幅 <0.1dB。
