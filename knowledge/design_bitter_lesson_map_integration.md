## 1. 审核结论(按严重度排序)

### 1.1 致命层:训练信号带宽比监督信号低约五个数量级

这是"从随机初始化直接 GRPO"必败的信息论直觉,独立于五个 log π bug:

- 判官对一组 n=4 条轨迹给全序,每组信息量上限 `log2(4!) ≈ 4.6 bit`。
  默认 8 组/run ⇒ **一整个 run 的监督信息 ≤ 37 bit**,却要从随机初始化更新
  数百万参数。且这 37 bit 还是上界——判官对纯噪声也编造全序
  (`tests/probe_judge_io_haiku.py`),真实互信息更低。
- 对比 BC:每 tick 监督 = 2 维相机(各 log2(11)≈3.5 bit)+ 20 键(各 ≤1 bit),
  量级 ~20 bit/tick;12800 tick ≈ **2.5×10⁵ bit/run**。差 4~5 个数量级。
- 序数优势还把这点信息进一步压扁:n 固定时 z 归一后的 adv 是**同一组固定数**,
  与轨迹真实差距无关(`knowledge/arch_current.md §5.3`)——等价于固定 margin 的
  轨迹级对比学习,梯度方差极大。

与仓库唯一的受控实证一致:GRPO 两次起效都从 **BC 暖启动**起跑
(`conclusion_fasttower_skill_ceiling.md`,可见目标 0.50→0.81)。
**判官排序的正确角色是在已有技能上做偏好精修,不是从零灌注技能。**

### 1.2 结构层:三处与"要学的映射"对抗的结构

1. **aim 是空间量,却走 FiLM 全局调制**。要学的核心映射是
   `目标像素 (x,y) → 相机增量 (dyaw,dpitch)`——一个近似线性的、平移等变的几何映射。
   现结构先把特征图 flatten 成 256 维(`net/pixel_tower.py:92`,空间结构毁掉),
   再用 (x,y) 做逐通道仿射(`:130`,与位置无关的全局乘加)。这是把一个网络
   **一层就能表示**的映射,改造成必须靠死记硬背查表才能逼近的映射。
2. **策略无任何跨帧记忆**。采样时 T=1(`grpo_pixel.py:236`),快塔是纯反应式单帧
   策略;即使修掉采样/更新失配、统一到 T=64,记忆上限也只有 64 tick=3.2 秒。
   "目标转出视野后找回来"、"探索未去过的方向"这类任务在该结构下**不可学**,
   与训练算法无关。用时间注意力窗口承担空间记忆并不划算——空间记忆应由
   显式地图承担(§3),时序骨干只需短窗口。
3. **20 键独立 Bernoulli + 零 bias 初始化**:起点期望每 tick 按 10 个键,
   "抽搐乱动"是结构必然。修法已登记(bias←logit(0.05)≈−2.9),属先验注入而非补丁。

### 1.3 喂数据层:五个 log π bug 之外的两个数据缺口

五个必修 bug(采样 T=1/更新 T=64、采样不关 dropout、温度只除采样侧、
goal_last 重放、每组 24 次无 ratio 的 opt.step)已在 `docs/next_session.md §3`
钉死,不重复论证。补两条核查发现:

- **goal 逐 tick 不可重建**:`goal_log` 只存 `(t, subgoal文本)`(`grpo_pixel.py:230`),
  **aim 没存**。goal 向量 = MiniLM(subgoal) ⊕ aim/1000,故即使想按 tick 回放也缺
  一半数据。修法:逐次慢塔刷新时把 `(t, subgoal, aim)` 全落盘(24 字节/秒,零成本)。
- **pose 已采而未用于任何学习**:`grpo_pixel.py:262` 已记录 `full.x/y/z`,
  当前只进判官证据文本。它是训练侧的免费标定/监督源(§3.3 用它标定里程计),
  但按纪律**不得进部署回路**。

## 2. 苦涩的教训:这仓库读对了一半

**读对的**:退役 4 类词表 / 凸包树干 GT / 手标分割头。实测词表把 YOLOE
90.8% 像素覆盖塌到 1.3%(`probe_yoloe_coverage`)——领域词表是典型的
"人把自己认为重要的知识编码进系统"。

**两处偏读,须纠正**:

1. **Bitter Lesson 的两个可扩展支柱是 learning(大数据)和 search(大交互)。**
   路线 1"从零 conv + 每组 4.6 bit 判官"两个支柱都没有:既抛弃了大规模预训练
   表征(learning 的杠杆),又没有密集环境信号(search 的杠杆)。它形式上"无先验",
   实质上落在"既不通用也不可扩展"的最差象限。Sutton 反对的是**编码领域内容**,
   不是反对**利用海量数据与算力得到的通用表征**——冻结 DINO-CLS 的 BC 学会
   aim+attack 全族(0.31–0.69)恰是通用表征有用的实证。VPT 视频 BC 暖启动
   正是 learning 支柱的落地方式,不是违背。
2. **编码"普适几何"的结构先验不在被反对之列。** CNN 编码平移等变、attention
   编码置换等变,Sutton 从未反对;被反对的是编码"树干长什么样"。北锚定地图把
   SE(2) 等变性写进坐标账本(整格 roll 无损、旋转推迟到读出),和 CNN 的权重共享
   是同一类东西:**结构承载几何(精确已知、普适),内容全部靠学。**
   因此 §3 的地图整合与苦涩的教训不冲突。

## 3. 下一版结构:提案 token + 北锚定地图 + goal-as-query

### 3.1 每 tick 的 token 集(替代 FiLM + flatten)

```
tokens_t = [ P_1..P_N ]  ⊕  [ M_1..M_K ]  ⊕  [ prev_act ]      (KV)
query_t  = goal_512 (+ 可学的策略 query 若干)                    (Q)

P_j = W_p · [ e_j(512, YOLOE 类别无关单位嵌入) , cx,cy,w,h,conf,area ]   N ≤ 256
M_k = W_m · [ map_feat(c维) , posenc(x_k,y_k) , level ]                   K ≈ 48–192
goal_512 = L2norm( W_g · [ MiniLM(subgoal)(384) , aim_x, aim_y ] )
```

- cross-attention 输出(几个 query 的读出拼接)→ 时序因果 transformer
  (短窗口即可,长程记忆归地图)→ cam_head(mu-law 11 bin CE)+
  key_head(20 Bernoulli,bias 先验注入)。动作头口径全部保持,已被裁决/验证。
- FiLM 删除。goal 从全局调制信号变成查询,空间选择由 attention 学——
  这正是路线 2 契约(`arch_current.md §1.2/§2.3`)的落地形态。
- **地形盲区补丁(可选低优先)**:提案 token 是物体中心的,地面/洞/沟不成"物体"。
  若闭环表现出摔洞/卡地形,加一组粗 patch 网格 token(如 6×10 的 stem 特征),
  不加手标类别——仍是类别无关的通用表征。

### 3.2 地图:EgoMapClip + 自定位(全部已过探针,首次接进策略)

选型直接采用已验证的组合(`docs/architectures/fovea-experiments-index.md M3-pre/M3-loc`):

- **EgoMapClip**(北锚定 + 3 级嵌套):75% 预算下近场持平均匀格、覆盖半径 2×,
  预登记 PASS。naive 逐步重采样臂中场 0.365 塌方,弃。
- **自定位**(观测 vs 存图互相关):漂移 0.33× dead-reckoning,预登记 PASS。
  MC 前提成立:yaw 由相机动作积分精确,漂移只在平移 → 匹配无旋转搜索维。

**写路径(每 tick,类别无关)**:

```
每个提案 j:  落地点 p_j = IPM(cx_j, cy_j+h_j/2)      # 框底中点,IPM 数学已验精确
             写入特征 f_j = W_c · e_j                # 512 → c(c≈8–16),W_c 可学
map.write(p_j, f_j)                                  # 双线性 splat,可微
```

不再写类 softmax 通道(旧 MapQuery 依赖 n_cls,已随词表退役)。
`W_c` 是唯一新增可学件,由读路径回传的梯度训练(读写均可微)。

**里程计与纪律**:部署回路 = 相机动作积分(yaw)+ 按键×标定步速(平移)
+ 周期 relocalize 修正。env 的 `full.x/y/z`(已在采,`grpo_pixel.py:262`)
只在训练侧做步速标定与漂移评测——特权信息只进训练侧,不进部署回路。

**读路径(每 tick)**:三级各池化成小 token 网格(如 4×4×3=48 个 M_k),
或近环密远环疏的 fovea 采样;带位置编码进 cross-attention 的 KV(§3.1)。
`goal_512` 与 `f_k` 经 `W_c` 同源,goal-query 对地图 token 的注意力天然是
"按语义找记忆里的东西"——**"目标在身后"从不可表示变成一次内积**。

**aim 钉(慢塔↔地图的免费联动)**:慢塔给的 aim 是屏幕坐标,零阶保持 20 tick,
转头即失效。把 aim 经 IPM 转成 ego 地面点、作为一个特殊 token 写进地图
(或直接作为独立 KV token 带衰减),goal 就获得跨帧持久性:转头后目标不丢。

**慢塔侧 MapQuery 修复**:`nearest(cls)` 改成 `nearest_by_embed(query_512)`
(goal_512 与格特征余弦,类别无关);`survey`(八扇区覆盖)本就类别无关,直接沿用,
且它是"宣告找村庄→探索覆盖须在涨"这类意图-行为一致性审计的免费进度计。

### 3.3 参数量级

地图非参数(状态张量,c=8/size=32/3级 ≈ 25k float 状态,零参数);
新增可学参数仅 W_p/W_m/W_g/W_c 四个投影 + cross-attention 块,≪1M。
计算 O(1)/tick(roll+splat+采样),20 Hz 无压力。

## 4. 喂数据重排(顺序即优先级)

1. **修五个 log π bug**(`next_session.md §3`,阻塞一切;含本文 §1.3 的
   goal 逐 tick 落盘 aim)。不修则任何对比实验都是噪声。
2. **BC 暖启动**——唯一被受控验证的 GRPO 起效前提。数据 = VPT 人类视频
   边缘化到 22 维契约(管线已冒烟,`vpt-teacher-minerl-free`);这是 learning
   支柱的落地,每 run 监督带宽从 37 bit 抬到 10⁵ bit 量级。地图模块在 BC 阶段
   即可接入(离线轨迹有动作序列可积分里程计),让 W_c/W_m 先学会读写。
3. **GRPO 精修**(判官排序,组内相对优势):
   - 采样=更新:同 T、`eval()` 采样、温度两侧一致、逐 tick goal 回放;
   - 一组窗口梯度累积单次 `opt.step`,或补 ratio+clip 成完整 GRPO;
   - 判官带宽:提高组数与每组条数比堆 tick 更值(信息 ∝ log(n!)·组数,
     与 tick 数无关);每 run 带退化输入对照臂测判官幻觉率;
   - 里程碑只作汇报锚点,手工统计量不进训练信号(纪律沿用)。

## 5. 可证伪预告(登记在先,防事后口径)

- 地图接入后,"目标出视野后重新捕获率"应显著高于无地图臂(同种子同预算);
  若无差异,地图读路径未被策略利用,查 attention 权重是否塌在 P token 上。
- goal-as-query 替代 FiLM 后,BC 的 cam 首步 acc 对 aim 的依赖应可测
  (置换 aim 应显著掉点);若置换不掉点,说明 aim 仍未进梯度。

## 7. 互斥修改路径与按苦涩教训的裁决

各错误给出的互斥路径(A 接口 / B aim 坐标系 / C 视觉前端 / D 时序输入 /
E 训练信号)完整讨论见会话记录;此处只记**按苦涩教训选定的组合**与落选原因:

- **A1 语言 token 直入 + hindsight relabeling**:快塔弃 MiniLM,直接吃 subgoal
  tokenized 文本(从零小嵌入表),grounding 靠 Omni 离线给 VPT 轨迹片段事后标注
  造 (指令,轨迹) 对——随算力/数据扩展。
  ⚠️ **撤回本会话早先的 A3(动词词表)推荐**:16 个动词 = 意图空间的 n_cls,
  与 4 类词表是同构的塌缩(512d→C 投影丢正交补),迟早重演退役。
  A2(传 Omni 隐向量)理论最优但需两塔联训,算力不支持,搁置。
- **B1 aim 钉进地图**:aim 仅在下发 tick 有效,IPM 转 ego 地面点写入北锚定地图,
  保鲜由坐标账本完成。落选:B2 快塔内部跟踪器(从零学跟踪本身成瓶颈,无出视野兜底)、
  B3 慢塔提频(治标,快转头下 4 tick 仍偏 ~70°)。已知缺陷:IPM 对空中目标(树冠)投错平面。
- **C3-DINO 稠密 patch 网格,替代 YOLOE(2026-07-10 后半用户直接拍板,原探针门控作废)**:YOLOE = DINO 级通用特征
  + **人工监督的 objectness 头**;删 n_cls 只删了一半人工先验,物体性先验仍在——
  地形/洞/坡不成"物体",在提案表示里结构性缺席(地形教训:洞 mIoU 0.156 佐证该盲区致命)。
  DINO patch 网格:语义等价或更强、**建图明确更强**(每个地面 patch 都可经 IPM 写图,
  地图从物体登记簿升级为稠密特征地面)、90×160 下 60 token 比 256 提案还便宜。
  两处已知短板及修法:实例离散性(让 attention 学,slot-attention 可学性已证)、
  stride-16 定位量化(**中心凹双尺度:全图粗网格 + 准星裁剪细网格,裁剪不缩放**——
  规避"2× 放大纹理尺度漂移"教训)。
  澄清:此前退役的是"冻结 DINO-CLS 单向量 BC"的用法,非 DINO 本身;
  "池化毁空间信息"教训(patch 互相关 0.27–0.45 vs 池化后 R²≈0)正面支持 patch 网格。
- **D1 帧堆叠(2–4 帧)**:同时消灭 T=1/T=64 失配 bug 与速度盲;长程记忆职责
  明确划给地图(空间)与慢塔(任务)。落选:D2 滚动窗+KV cache(实现重,pos 要改相对式)、
  D3 从零递归态(收益未证,BPTT 重)。
- **E1 BC 暖启动为主信号**(VPT 边缘化 22 键契约,管线已冒烟)→ GRPO 只做精修;
  判官全量落盘攒 RM(E3)为后手。落选:E2 纯 RL + 判官协议改造(分段排序×加组数
  只抬一个量级,离 BC 差三个;仅当"证明纯 RL 可行"本身是研究目标才走)。

**目标结构(v2 最终形态)**:

```
每 tick:
  DINOv3 patch 网格(全图粗 + 准星 fovea 细,~120 token,冻结骨干 + 可学投影)
⊕ 地图 token(EgoMapClip 三级读出 ~48 token;写入 = DINO patch 经 IPM 稠密落地)

离线轨迹可积分里程计)→ GRPO 精修。
```

应急备选(不动大盘,~100 行改动,仅当"先出一次可解释训练"优先时用):
aim 画 2D 高斯热图当第 4 通道进 conv stem(C2)+ 帧堆叠 + BC 暖启动 + 慢塔提频。

## 8. 预登记裁决探针:DINO vs YOLOE(执行顺位最高的新实验)

> ⚠️ **本节作废(2026-07-10 后半,用户直接拍板)**:视觉前端定 DINO,YOLOE 整线废弃
> 并已删码,双臂对比不再需要。探针降级为 **DINO 单臂瞄准可学性验证**
> (`tests/probe_dino_aim.py`:R² 显著>0 且地形分层不塌;FAIL 则跑 fovea 双尺度臂)。
> 下文保留作预登记历史记录。

## 9. 慢塔会话契约(Mamba 固定态判决)

**判决:"无限上下文"是吞吐性质不是记忆性质。** 固定潜向量 = 容量有界 +

三层否决无限单会话(设计 1):

(b) 文献:纯 SSM 精确关联召回系统性弱(Zoology / "Repeat After Me"),
**Nemotron 做成混合架构本身 = 设计者对纯态召回的对冲**;
(c) 工程:状态不可观测、不可回滚;混合架构注意力层 KV 仍线性涨,无限流物理不成立。

**选定:设计 2(无状态重提示 + 状态全外置)为主线**;设计 3(30–60s 滑窗会话

prompt 契约(每次调用,固定前缀可被 vLLM prefix cache 命中):

```

  decision(继续|换目标|重规划,微决策词表已验证 acc 0.937) /
  subgoal(≤6 词) / aim([X,Y] 0..1000) / done_when(≤8 词机器可核对完成判据)
[user 每次重建] 状态行(t/库存/位移)+ 最近子目标及达成标记 +
```

闭合的断边:prev_done+decision = 快塔→慢塔上行 + 终止语义;done_when = 判官

留出召回探针验过 → 才允许把地图/状态行部分职能移交给模型内部状态。

## 11. 多游戏泛化与自标定(2026-07-10 用户裁决:少提前假设,参数不绑死)

### 11.1 自标定原则(net/calibration.py,已实现)

**分辨率/FOV/相机增益/步速/键位语义是环境的参数,不是代码常量。** VPT 是 px 单位、
不同游戏不同 FOV、键位可任意重映射——把它们写死(CAM_MAX_DEG/CAMERA_SCALE/固定
fov_y)就是把训练集参数固化进模型,此类错误已发生过一次(相机单位 45% 削顶,见记忆
colab-camera-units)。正确形态:**智能体用已知动作当探针,从光流响应测出参数**:

- 在线(纯观测,可进部署回路):开局 `probe_plan`(小角对称对,净漂移 0)→
  `flow_shift` 相位相关 → px/deg 增益 → FOV=H/gain;`calibrate()` 已接进 rollout。
- 离线(数据集,键位无关):`fit_action_flow_map` 对 (帧对, 动作向量) 做
  flow≈M·action 最小二乘——**不需要知道动作通道叫什么**,相机通道自然得大系数、
  增益即标定;VPT/gaming500/任意数据集同一套代码。
- 步速:当前用 env pose(特权,只进训练侧,产出常量随 SelfCalib 状态携带);
  升级路径=视觉里程计。
- 产出三接口:`physics_line()`(进慢塔 prompt,未标定如实说 uncalibrated 不编数)、
  `physics_vector()`(进 token 塔 geo,带有效位)、`fov_y_deg`(进 IPM/地图,接线时替固定 70°)。

### 11.2 慢塔 I/O 结构与提示词(游戏无关化,已实现)

system 提示词零领域知识;每次调用的 user 内容 = 四行结构化输入 + 帧:
`TASK`(长程目标,唯一领域知识载体,--task 可换)/ `STATE`(时间/资源/位移/
最近子目标及 done 标记)/ `PHYSICS`(自标定结果)/(接线后)`MAP`(survey/最近匹配)。
输出五字段 JSON 不变(prev_done/decision/subgoal/aim/done_when)——字段本身游戏无关。
换游戏 = 换 TASK 行 + 重跑自标定,契约与代码不动。

### 11.3 慢塔思维模式微调方案(设计,待数据攒够)

底座配方沿用已验证 LoRA(r16/qkvo/lr1e-4,新 adapter 防复核回退)。三阶段,
每阶段的数据都由现有管线自动产生,不手标:

1. **拒绝采样 SFT(决策质量)**:设计 2 会话日志 (TASK,STATE,PHYSICS,帧 → 回复) 中,
   只保留"subgoal 后来被机器核对达成"的回复(done_when 是机器可查条件 ⇒ 达成判定
   自动化;prev_done 字段直接有真值标签)。**思维模式不是人写模板教的,是被
   "成功的自己"筛出来的**——苦涩教训口径:淘汰式塑形,不注入手工推理链。
