# 项目知识库

> 本文件是仓库的唯一现行知识文档。代码是逻辑事实来源；本文只保留跨模块设计、已经改变决策的实验结论、运行约束和停止规则。历史过程、失效方案、逐次训练日志、原始探针和可从代码直接读出的微观参数不在此重复，必要时查 Git 历史。

## 1. 项目定位与现行主线

项目研究从游戏交互中迁移控制能力。游戏是训练载体，不是最终产品。现行可运行主线是 CraftGround 上的 GRPO-Pixel 双塔策略：

- 快塔以游戏帧、上一动作和慢塔目标为条件，按 tick 输出相机与按键动作。
- 慢塔以较低频率读取画面和状态摘要，输出文本子目标与 `0..1000` 归一化像素指点。它负责语义规划，不直接闭环控制。
- VPT 人类录像行为克隆提供动作先验；可选的 VPT 教师反向 KL 强化持续攻击等稀缺行为。
- 判官只比较同组轨迹的相对进展。当前协议是双向成对比较、一致性门和 Copeland 计分；无充分证据时必须并列。
- 早期 Δz-JEPA、RSSM、Dreamer、PPO+AD、YOLOE 和 Mamba-belief 路线已退役。可复用结论已压缩到本文，不再把这些路线当待办。

### 1.1 两个快塔版本

| 路径 | 输入与结构 | 当前用途 | 已知缺口 |
|---|---|---|---|
| v1，默认 | 四帧像素堆叠，经可训练卷积干与因果注意力；MiniLM 目标向量以 FiLM 注入 | VPT BC、教师蒸馏和 GRPO 的成熟路径 | 全局压缩弱化空间结构；目标措辞偏离 hindsight 训练分布时控制力下降 |
| v2，旧原型 | 冻结 DINO patch、北锚定地图 token、UTF-8 子目标、几何量与上一动作，经单层 goal-as-query cross-attention | 用于验证 DINO、地图和 token 回放接线 | 缺少预训练语言语义和真实时序混合；地图写读没有同图梯度，不作为下一版主干 |

两路共享动作契约：相机两轴使用 11 档 mu-law 分类，20 个按键独立 Bernoulli，反应式动作 `chunk_k=1`。v1 与 v2 checkpoint 不兼容，必须分开保存和加载。

### 1.2 v2 目标架构

新 v2 定义为语言条件化的时空 DINO 策略。它不兼容旧 v2 checkpoint。核心主干必须在关闭地图时独立完成闭环控制，地图和其他长期记忆只能通过 `MemoryProvider` 作为可选 token 注入。

```text
当前帧 288×512 → 冻结 DINO → 18×32 高分辨率 patch ─┐
历史帧高分辨率缓存 → 2×2 token pooling → 9×16 时序 ─┤
                 → 因子化空间/时间混合 ─────────────┤
文本 → 冻结预训练文本编码器 → 缓存文本 token ───┤
像素指点 → 相对 patch 位置偏置 ─────────────────┤
动作历史 + dt + 自标定有效量 ────────────────────┤
可选 MemoryProvider ────────────────────────────┤
                                                 ↓
                                    目标条件 action queries
                                                 ↓
                                      结构化物理动作分布
```

- 文本编码器在慢塔目标刷新时运行并缓存。默认复用已有 MiniLM 语义空间；不得再让小规模控制数据从 UTF-8 字节重新学习词义。完整文本 token 和汇总 token 同时进入策略；目标汇总还条件化每个时空块，避免语言只作为一组可忽略 KV。
- DINO 默认读取 `288×512` 当前帧，形成 `18×32=576` 个 patch。动作 query 读取当前帧的完整网格；历史帧沿用其首次编码的高分辨率 token，但经 `2×2` 空间池化为 `9×16` 后参与跨帧混合。禁止把旧原型的 `96×160`、`6×10` 网格作为生产默认，也不使用 CLS 或全局池化代替空间读取。训练必须保留连续序列维，禁止把监督 tick 展平成 `T=1` 后宣称存在时序注意力。
- 若 `288×512` 不能满足 20 TPS 的端到端 p95 延迟门，不直接退回缩略图。先评估冻结特征缓存、减少历史混合频率和蒸馏轻量视觉前端；分辨率调整必须以细目标闭环非劣为前提。aim 中心高分辨率裁剪只作为消融项，不能替代全画面上下文。
- 像素指点不只拼进全局向量。它生成 patch 相对位置偏置，并作为独立 query 条件，使改变 aim 能直接改变视觉读取位置。
- 默认记忆实现是 `NullMemory`。现有地图只有在同一 episode 连续重放、写入与读取处于同一计算图且通过 memory-on/off 闭环消融后，才能升级为默认能力。
- 输出仍提供旧接口的相机 `2×11` mu-law 分布和 20 键视图，但真实采样分布改为结构化动作：两轴 look、两轴模拟移动、姿态、互斥 hotbar 与事件/持续按钮。20 键由该分布确定性展开，禁止同时采样前后、左右、潜行与冲刺或多个 hotbar。
- 设备差异只进入执行适配器。策略输出参考周期下的物理角速度、移动意图和按钮语义；键鼠适配器换算鼠标增量与键位，手柄适配器按标定响应曲线换算双摇杆和按钮。`dt`、请求动作、实际执行动作和生效延迟分别记录；GUI 光标与世界相机使用不同校准。

新 v2 的目标规模是约 `4–6M` 可训练参数，不含冻结的 DINO 与文本编码器。默认训练和在线推理使用 BF16，几何、地图递归状态、损失归约和优化器状态使用 FP32。NVFP4 只保留为 Blackwell 上的显式实验模式：先评估冻结 DINO 或较大的线性投影；LayerNorm、softmax、激活、动作头、几何输入和递归状态不得量化。在线端到端 p95 延迟未降低至少 `10%`，或训练更新时间未降低至少 `15%`，即保持 BF16。

### 1.3 数据与控制流

1. CraftGround 产生 RGB、完整观测和 V2 动作接口。
2. 自标定从可观测运动估计相机旋向、增益、FOV、步速和延迟。无法测得的量保持无效，不填猜测值；依赖它的写图、钉点或里程计显式降级。
3. 慢塔周期性产生子目标和像素指点，快塔在两次刷新之间零阶保持这些条件。
4. v1 使用句向量与像素帧；v2 使用 DINO patch、地图、字节语言和几何 token。
5. rollout 记录每 tick 的目标、上一动作、采样分布所需输入和结果；更新必须在同一输入、温度与网络分布上重算 `log π(a)`。
6. 同一世界起点的多条轨迹进入成对判官。组内相对优势驱动一次 on-policy REINFORCE 更新；当前实现没有 GRPO 的 ratio、clip 或 KL，因此准确称为“组内相对优势 REINFORCE”。

## 2. 不随实验变动的设计约束

### 2.1 表征与先验

- 优先使用可扩展的通用预训练表征和数据，不为单个游戏手写类别词表、分割头、凸包真值或调试屏解析器。
- “不写领域补丁”不等于从零学习全部感知。冻结 DINO patch 和 VPT 人类先验符合当前取舍；人工 objectness、类别压缩和特权教师直接进部署策略不符合。
- 保留 patch 网格的空间结构。CLS、全局池化或低维 PCA 只能用于已证明不丢目标信息的场景，不能作为“信息不存在”的判据。
- 长程任务状态归地图和慢塔，短程速度由多帧观测承担。单帧策略不能辨认自身速度、目标接近速度和相机惯性。
- 慢塔意图通过显式语言 token 传递，不读取预训练模型内部递归状态。旧受控实验中文本接口的反事实服从率为 `0.940`，而递归状态未保留视觉 age。

### 2.2 训练与评价

- BC 是当前策略产生有效动作的前置；GRPO 只强化策略已经有非零概率产生、且轨迹中可被观察和比较的行为。
- raycast、环境位姿、heightmap 和命令注入只可用于训练侧标定、课程与评测，不能进入部署观测。
- 离线 loss、动作准确率和教师一致率只作过程指标。最终验收必须是固定协议、固定 seed 集上的闭环成功率与任务事件。
- 长程正式验收使用自然生成世界、随机 seed、冷启动、无外部教师与部署特权，样本量至少 `n=16`；报告成功率、置信区间和中位耗时。
- 手工里程碑用于报告和审计，不作为主训练信号。判官必须引用帧号或数值证据；图文冲突以画面为准；死亡立即截断并写入证据。
- 判官全并列时优势为零并跳过更新。旧的强制全排序数据不得与 `pairwise_v2` 混用训练本地奖励模型。
- 采样与更新必须使用相同温度、相同目标、相同上一动作和等价网络。策略 dropout 保持为零；一组轨迹累积梯度后只执行一次优化步。
- 教师必须是学生观测的确定性函数。任何升级都要先给出上一级在固定预算内的证伪结果。
- 判官调用必须全量落盘。每个判官驱动实验都要包含退化输入对照，并检查 `metrics.jsonl` 的 `fallback`；判官图片必须位于工作区内。

### 2.3 多环境与持久状态

- 分辨率、FOV、相机增益、步速与延迟是环境状态，不是模型常量。
- 当前快塔动作是固定 tick 的增量，模型没有 `dt` 输入。跨设备部署必须保持固定执行节拍；若允许变帧率，需要显式加入 `dt` 和跳帧增广。
- 地图以北向世界坐标写入，以当前视角读取。部署侧里程计只能使用动作命令和自标定结果；环境真值只用于校验。
- 首木 GRPO 起效后才启动深阶段课程。课程起点可用 `give/tp` 构造近似状态；需要精确过程态时使用完整世界快照或“seed + 动作序列重放”，并先验证引擎确定性。
- 并行环境必须使用独立的 MinecraftEnv 目录、端口和 `run/saves`，禁止在 JVM 仍写盘时复制世界。

### 2.4 数据边界

- 第一阶段只使用同域 VPT 数据。训练池滚动读取 all_6xx/8xx/9xx/10xx_Jun_29；holdout 固定为 all_7xx_Apr_6、3 段、seed 42，与训练索引族隔离。
- gaming500 只可能作为第二阶段预训练语料。启用门槛是同域 VPT 达千段量级、holdout 已平台化且模型容量已扩大；不得混入第一阶段 BC。IDM 反标网络视频不再采用。
- 可发布教师标签、词表和 manifest；不再分发原始 VPT 录像，因为原数据没有明确再分发许可且含 Minecraft 知识产权内容。

## 3. 已改变决策的证据

这里只保留会影响架构、训练顺序或资源选择的最小证据。逐样本结果和旧探针可从 Git 历史恢复。

| 结论 | 最小证据 | 决策 |
|---|---|---|
| DINO patch 足以承载瞄准信号 | 活环境 104 样本、5 折 `R²=0.899`；hole/slope 分层均约 `0.88` | v2 使用冻结 DINO patch；不触发双尺度 fovea |
| VPT 动作契约与 CraftGround 环境可闭环 | VPT 教师 joint 与 marginal 两种翻译各 `5/5` 获得原木 | 当前失败归因于学生策略和数据，不重写动作契约 |
| hindsight goal 确实被模型使用 | 真 goal 对乱序 goal 的 loss 差 `-0.0096`，95% CI `[-0.0130,-0.0062]`；zero goal 比 canonical 低 `3.64%` | 保留 hindsight relabel；慢塔措辞应贴近训练词表 |
| 目标通道已有行为功率，瓶颈转向 aim | zero/explore/固定词表/真慢塔/阈值投影的 attack duty 为 `0.17/0.21/0.55/0.56/0.58`；固定词表首次 `1/5` 取木，其余学生臂为 `0/5` | 慢塔输出使用 `<verb> <item>` 训练语言；下一实验优先瞄准跟随 |
| GRPO 依赖可见奖励信号与 BC 暖启动 | 可见木头成功率 `0.50→0.81`；视觉不可区分的石头任务 `0.31→0.06` | 先解决可学性与起始策略，再扩大 on-policy 预算 |
| 当前 GRPO 已有排序信号但仍未完成任务 | 词表目标 + bc_vpt4 的 `4×6×1200` pairwise_v2 无 fallback、环或调用失败，产生非零梯度，但木头里程碑仍为 `0` | 不再用更长 tick 证明信号存在；先修 aim |
| 平均 BC 不适合稀疏、精确、零容差 GUI 行为 | holdout loss `0.0013` 时合成闭环仍为 `0.00`，教师为 `1.00` | GUI/背包/合成等稀疏键交给慢塔或专门模态，不以总体 BC loss 判定 |
| 反应式策略不应动作分块 | 首步相机准确率 k1/k4/k8 为 `0.275/0.270/0.256`，闭环切换率在 k8 为 `0` | 固定 `chunk_k=1` |
| Omni 可作慢塔，不能零样本作控制器 | RTX 5090 上权重约 `21.5 GiB`、热图像 TTFT `0.154 s`；40 次决策中直控放置 3 块且动作仅 1 种，脚本基线 39 块 | Omni 只做低频语义与指点，快塔负责连续闭环 |
| 慢塔是可替换件 | 同口径 32 题两塔四项合规均 `100%`；Omni 延迟 p50/p95 `0.320/0.363 s`，Qwen 为 `1.485/1.601 s` | 5090 优先 Omni NVFP4；约 24G、sm_89 机器使用 Qwen FP8 |
| Omni 历史必须有界 | 当前服务栈不支持 Mamba prefix cache，每轮完整 prefill；4/16/40 帧 TTFT 为 `0.143/0.389/0.823 s` | 外置状态并使用有界滑窗，不把无限视觉历史灌入慢塔 |
| 慢塔有树干定位信号但不足以承担控制 | 同 seed 各 16 次，恒定/随机/慢塔的 tree-hit 为 `0.250/0.438/0.312`，但 trunk-hit 为 `0/0/0.312` | 保留像素指点作条件，不能用宽松 tree-hit 证明闭环控制能力 |
| 摘要判官不能识别真实细微差异 | 24 题总体 `0.833`，损坏与原型题 `20/20`，真实差异题 `0/4` | 必须看画面、引用客观证据并默认并列；不以总体准确率训练本地 RM |
| 渲染路径取决于是否与训练抢 GPU | 无争抢 ZEROCOPY `107.2 sps`；同卡并发时 Xorg+RAW `37.3 sps` 对 ZEROCOPY `25.3 sps` | 采集独占 GPU 用 ZEROCOPY；训练同卡并发用 Xorg+RAW |
| 全局深度和升分辨率不是可靠的细地形对策 | 洞穴判别 raw depth mIoU `0.156`，反投影高度图 `0.807`；2× 推理分辨率 mIoU `0.530→0.248` | 用几何反投影，不用盲目放大或原始深度阈值 |

## 4. 明确停止的路线

以下不是“以后再试”的候选。除非出现新的可证伪假设和不同实验条件，否则不要重复投入：

- 不用手工统计量训练或证明学习发生。恒定复读与目标策略曾同分，随机策略还可能更高。
- 不把 LLM 判官的强制全序当证据。它会对无意义图像编造差异；必须使用默认并列、双向一致性与客观举证。
- 不用离线准确率或 BC loss 代替闭环验收。同一离线水平对应的闭环成功率可从 `0.06` 到 `0.81`。
- 不蒸馏学生观测不到的特权教师，也不蒸馏含随机潜变量的动作；同一画面对应冲突标签会把动作均值推回零。
- 不让预训练 LLM 的递归状态兼职视觉世界记忆。既有 LoRA 与递归张量解冻实验未恢复视觉 age 信息。
- 不用人类录像的单帧全局相位相关标相机。HUD 和运动模糊曾使增益衰减约 7 倍。
- 不读取 F3 等调试屏特权信息作为部署标定或监督。
- 不使用游戏类别词表、手标分割头、凸包树干或 YOLOE objectness 补丁。词表曾把像素覆盖从 `90.8%` 压到 `1.3%`。
- 不用池化加 PCA 的代理实验否定完整 patch 网格中的信息。
- 不用无上限 OHEM 或自身误差重加权；它破坏过流匹配收敛，已有开环下降 `4.10 dB` 的结果。
- 不在目标视觉不可辨、策略没有 BC 起点时继续加 GRPO 时长。先修观测、数据和行为支持集。
- 不把零样本 VLM 当连续相机控制器。连续符号和增益必须由专门闭环与物理标定承担。

## 5. 当前实现边界与下一顺序

### 5.1 已接通

- v1：VPT 数据流、hindsight relabel、BC 暖启动、可选 VPT 教师标签与反向 KL、慢塔目标、GRPO rollout/update、pairwise_v2 判官。
- v2：DINO 前端、DINO patch 多帧身份、EgoMapClip 写读、aim 钉点、UTF-8 语言 token、几何有效位、GRPO 采样与回放、独立 checkpoint。
- CraftGround：死亡截断、有树 seed 筛选、完整世界快照、V2 动作日志、逐 tick 状态指纹和冷启动重放。

### 5.2 尚未接通或未完成验收

1. 约 50 帧活环境树干真值上做慢塔 aim A/B：同帧同提示比较像素误差 p50/p90；行为臂同 seed 只换慢塔，比较近树与树叶接触。p50 差超过 2 倍才判能力差，否则按部署条件选型。
2. 为学生补 aim 跟随监督或相机蒸馏，再以固定 seed 闭环取木验收。当前无需继续证明 attack 通道或判官信号存在。
3. 以 §1.2 替换旧 v2：先实现 `NullMemory` 下的预训练文本、时空 DINO、aim 位置偏置与结构化动作，再做同图序列重放和地图消融；地图不得阻塞无记忆基线。
4. 扩大滚动 VPT 池后，从 bc_vpt4 checkpoint 继续教师 KL。小池对照只证明接线方向正确，行为会退化；是否有效以学生闭环 `got_log>0` 为准。
5. 同域数据扩大后，再依次消融相机两级头、稀有事件窗口采样和封顶的按键正例权重。一次只改一个变量。
6. aim 修复后再扩大 GRPO 组数；组数优先于单轨迹长度。
7. 本地奖励模型只在积累至少千级、经过 `pairwise_v2` 客观锚点过滤的有效对后启动；先用冻结 DINO 与小打分头，留出真实差异准确率不够再考虑慢塔 LoRA。

## 6. 安装与运行

### 6.1 核心环境

```bash
python install_env.py
# 或
pip install -e .
uv pip install -e .[dev]
```

Python 要求以 `pyproject.toml` 为准。DINOv3 权重受访问控制，需要 `HF_TOKEN`；无 token 时可显式选择开放的 DINOv2。所有数据、模型、快照和日志放在 `runs/`。

CraftGround 首次构建需要完整 JDK 21，而非只含 headless 运行时的裁剪包；Linux 还需要 GL、GLEW、GLU 和 GLFW 开发库。启动时确保 `JAVA_HOME` 指向 JDK 21。当前 `grpo_pixel.py` 还直接导入 `openai` 与 `sentence-transformers`，但它们未写入 `pyproject.toml`；环境脚本或依赖声明修复前需显式安装，不能假定核心安装已经包含。

### 6.2 慢塔选择

- RTX 5090 / sm_120 / 32G：使用 `tests/serve_omni_nvfp4.sh`。脚本已包含当前 vLLM/CUDA 兼容处理。
- 约 24G / sm_89：使用 Qwen3-VL-8B-Instruct-FP8，并控制显存占用。
- Qwen FP8 在 sm_120 上若命中 DeepGEMM 布局断言，设置 `VLLM_USE_DEEP_GEMM=0`；L4 上若 FlashInfer sampler JIT 失败，设置 `VLLM_USE_FLASHINFER_SAMPLER=0`。
- 判官需要 `claude` CLI。传给判官的图片必须位于工作区内，否则会静默进入 fallback，实验无效。
- 当前 GRPO 主循环显式使用 RAW 渲染；这与“独占采集 GPU 时 ZEROCOPY 更快”的基准结论不冲突。

### 6.3 最小命令

## 7. 世界快照与动作重放

CraftGround 没有 JVM 内存快照接口。项目保存的是落盘后的完整世界目录：先执行 `save-all flush`，推进一个 tick，再归档 `level.dat`、区块、实体与玩家数据；排除进程锁 `session.lock`。快照创建后不可原地覆盖，并用逐文件 SHA-256 校验。

轨迹保存完整 V2 键集合和相机俯仰/水平增量，不保存可能漂移的离散动作编号。每个动作后计算状态指纹，覆盖位姿、生命、饥饿、经验、时间、天气、选中栏与库存；RGB 不入指纹，避免渲染噪声造成假分叉。恢复时必须关闭旧环境、复制快照到独占工作目录、冷启动 JVM，并在首个不一致 tick 抛出 `ReplayDivergence`。

标准流程：捕获运行中世界并取得 flush 后观测；从该观测开始记录动作；关闭旧环境；恢复工作副本；冷启动并逐 tick 校验。实验若依赖特定方块或实体，应把对应字段加入完整观测和指纹，而不是绕过一致性检查。
