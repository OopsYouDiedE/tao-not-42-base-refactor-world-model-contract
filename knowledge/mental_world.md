## 0. 一句话立场

把系统从"看视频→反应"改造成**脑内世界**：脑子里维护一组**有限的持久潜向量**，它本身就是一个能自转的生成模型；
眼睛**只在偶尔扫一眼时注入预测误差**去纠正它。框架转向 **Transformer**，**弃用 Mamba**。

弃用 Mamba 的理由：当初上 Mamba 的唯一动机是"逐像素 SSM 建立不了运动对应"，其前提是"世界状态 = 像素网格特征图"。
一旦核心状态改为**有限抽象潜向量**（N 很小），该前提消失，Transformer 的全局注意力在小 token 集上更自然，
Mamba 的长序列线性扫描优势也用不上。

---

## 1. 核心表示：有限持久潜向量

- 世界状态 `Z ∈ [B, N, d]`：N 个**跨时间持续存在的潜向量**，不是稠密像素场。以最少数量拟合任务目标。
- **残差/渐进分解（非物体中心 slot）**：slot1 主成分、slot2 补 slot1 漏的、slot3 再校正……每个潜向量装"前面没装下的"。由此带来四条约束：
  - 天然有**幅度谱**（slot1 大 → slotN 小）；slot 间是**顺序正交（Gram-Schmidt 式）**，不是两两对称互斥；
  - **无实体身份可匹配**——Sinkhorn/匈牙利那套 slot 分派**不适用**（那是物体中心 slot 的解药）；
  - 重要性排序用 **nested dropout / Ordered Autoencoder**（随机只留前 k 个 slot 解码、要求仍能重建）诱导，不手搓正交损失；
  - ⚠️ **SIGReg 接线戒律**：勿把 `Z` 摊成 `[1, B*N, d]` 做池化 SIGReg——大小尺度混合在 SIGReg 眼里是**非高斯（峰态异常）**，会施压**碾平幅度谱**。须**方向/幅度拆开**：对方向（单位化后）防坍缩，让幅度自由成谱（详见 §7）。
- "每次更新完全保持潜向量"= **结构上**潜向量身份/数量不变（持久）+ **算子上**每步只做受限增益的残差改写（非扩张，I5）。
- 范式：**预测编码 / 主动推断**，落成 **predict–correct 两相**（类 Kalman / RSSM）。

## 2. 与 LeWM 的区别（本设计的身份证）

本设计**借用** LeWM（github.com/lucas-maes/le-wm）的内核，但在六处刻意偏离。每次"这不就是 LeWM 吗"的疑问，大概率撞上了共享内核——撞对了。

**共享内核（= LeWM，已验证，照搬不创新）**：JEPA 潜空间预测（不解码回像素）+ SIGReg 防坍缩 + Transformer + 全观测提供稠密 target。

**六条真区别（我们的下注，未验证，分阶段逐条验证）**：

| # | 维度 | LeWM | 本设计 |
|---|---|---|---|
| 1 | **读取** | 每帧全读、全帧全 ViT | **选择性读**：中央凹（空间稀疏）+ 偶尔睁眼（时间稀疏），靠蒸馏逼近全读 |
| 2 | **状态** | 每帧一 token 的滑动窗 | **N 个持久残差潜向量**（渐进分解，非物体 slot），就地改写（RSSM 式信念） |
| 3 | **索引** | 严格帧序号、规则稠密 | **(时间, 空间) 标签**，稀疏不规则事件 |
| 4 | **输出** | 只出下一帧嵌入 | 世界信念 Z（探针可读）+ **动作潜向量→定时动作序列**（onset/键/MosPos）+ gaze/wake |
| 5 | **可控性** | 隐式动作条件 | **显式可控闸门 c**（逆动力学接地）+ 不确定 σ |
| 6 | **用途** | 给规划用的世界模型（试动作 MPC） | 快速迁移底座 + 实时指导 + 推理期反应式策略，世界模型退训练期 |

注意 SIGReg **不在**区别里——它是共享的，我们借的。区别全在"怎么读、状态长啥样、输出什么、用途"，不在防坍缩。
"部分读取≈全部读取"正是第 1 条：老师（全观测 target）是 LeWM 式的，学生（部分读取）不是，故似曾相识又不是。

## 3. 闭环结构（愿景；活模型对齐见 [net/world_model.py](../net/world_model.py)）

```
                 ┌──────────────── 每帧 predict（脑内自转，可无输入）─────────────┐
img ──┬─ PeripheralVision ─→ M 个余光 token ┐                                     │
      │   (全帧低清, 便宜, 常开)            ├─ SlotBinder ─→ Z（门控残差注入, I5） │
      └─ crop_fovea(gaze) ─ FoveatedVision ─┘   = correct/纠错                    │
          (gaze 裁剪的高清 ViT, 贵, 偶尔)                                          │
                                                                                  ▼
   [Z, h(假说), a_tokens(动作), τ(Δt), u(动作查询), g(注视), w(唤醒)] ─→ WorldTransformerBlock×L
                                                                                  │
   解码头：StateDecoder(μ,σ,exist 概率云) · action_plan(定时动作) · gaze(看哪) · wake(多久后再看)
```

要点：

- **双通道感知 = 余光 + 中央凹**。余光（`PeripheralVision`）廉价常开、给全局显著性；中央凹（`FoveatedVision`）只对 `gaze` 指向的局部裁剪做高清 ViT。这就是"偶尔扫一眼"的物理实现——**不把整帧高清灌进脑子**。
- **correct = `SlotBinder`**：感知 token 经 cross-attn + 门控残差（增益 sigmoid 限幅，I5）注入 Z。"只动误差部分"由注意力自行定位 + 门控残差涌现，**不手工挑**该改哪个槽。`compete=True`（Minecraft）用 `SlotCompetitiveAttn`：softmax 沿 **slot 维**做零和竞争——一个 patch 的注意力质量被各槽瓜分，从结构上抑制多槽冗余。残余的"多槽盯同一显著物体"再由训练侧 **`slot_diversity_loss`（`--beta_div`）软惩罚竞争注意力图的成对重叠**收敛：约束加在**注意力空间(谁看哪里)**、前向注意力仍是合法分布(非负、沿 token 和 1)。**不**在前向里硬正交化(施密特那样硬改聚合权重会打破 `out=w·v` 的加权平均语义、并按 slot 序饿死后排槽,把"都看同一物体"换成"首槽看、后排饿死")。
- **predict = `WorldTransformerBlock`**：双向无掩码 Transformer 一次推演**全部** token。
- **异步节律**：`wake` 头输出 `T_wake`（多久后再睁眼），`τ = ContinuousTimeEncoding(Δt)` 把"距上次观测的时长"喂进推演——稀疏/不规则观测靠 Δt 兜底，没看清 ≈ null token。
- **主动感知出口 = `gaze`**：决定下一眼看哪 `(g_x, g_y, g_s)`，经 `crop_fovea` 的 **STN（affine_grid + grid_sample）可导** —— 这一点很关键：**"看哪里"可以被梯度直接学**，不必退回高方差的硬注意力 RL。
- **时空标签编码**：观测/查询 token 的"何时、何地"——时间用 `ContinuousTimeEncoding`（相对 Δt 正弦；⚠️ Δt 应以**帧**为单位喂入，否则正弦量程错配、低频通道失效）；空间用 `SpatialPosEmbed`（gaze 中心 `(g_x,g_y)` + 尺度 `g_s` 的 Fourier 特征→投影，**已注入 `e_fov`**，补上"脑子拿到高清却不知方位"的缺口）。未来**查询 token** = 学习 mask token + 时空标签寻址 + 按 `c` 注入随机（可控维确定预测、不可控维采样；不是字面零，也不是无约束噪声）。
- **plan-as-vector = `action_plan`**：DETR 式 K 个动作查询，各解码出 (按哪键, onset Δt, 时长, 是否真有)，一次输出一段**定时动作序列**而非单步动作。对音游这类"未来谱面已知"的任务尤其贴合。

## 4. 不可控性 / 不确定性

- `StateDecoder` 的 `σ` 承载 **aleatoric（不可消除）不确定**：不可控分量（谱面外 RNG、对手）方差大、不强求点预测；可控分量方差小、要求准。
- **待建：显式可控闸门 `c_i ∈ [0,1]`**（每个潜向量自带的可学坐标）：
  ① 动作只乘 `c` 流向可控分量；② 噪声乘 `(1−c)`；③ `c` 由**逆动力学头**接地（动作能解释某槽多少变化，它就多可控）。`c` 过 sigmoid 天然有界（I3）。
- 在 `c` 落地前，`σ` 暂时独自承担"不可控 = 高方差"的角色。

## 5. 训练设计（本文档重点）

**全程无监督 / 自监督**（无人工标注）：所有 target 都是**自生成**的——预测 target 来自模型自己编码的未来帧；动作 target 来自**训练环境免费暴露的谱面**（`env.get_upcoming_actions`，是游戏自带状态而非人工标注，故仍属自监督）；gaze/wake 的 target 来自模型**自身预测误差**（见 §5.6）。

整套损失要让四件事成立：

| # | 要成立的事 | 由谁保证 | 归预训练 loss |
|---|---|---|---|
| 1 | 潜向量不坍缩 | SIGReg | ✅ |
| 2 | 动力学能预测未来 | 预测损失 | ✅ |
| 3 | "可控"真的可控 | 逆动力学 | ✅ |
| 4 | 打得好 / 贴近完美序列 | 动作规划头（训练环境谱面自监督）/ 运行时 goal-cost | 训练✅ / 部署❌ |

**第 4 件分两种情形**：在**训练环境**里谱面可得（`ProceduralRhythmEnv`），动作规划**可**自监督预训练（代码已用 `gaussian_action_loss` / `action_plan_loss`）；在**部署的陌生游戏**里谱面不暴露，"完美序列"退化为运行时让 System-2 想象去够的 goal-cost，不是回归目标。

### 5.1 主动感知（active sensing）训练循环

1. `t=0` 全量编码首帧 → `Z_0`（有根）。
2. 不看，predict 滚 k 帧 → `Z_k^prior`（纯想象）。
3. 真实第 k 帧也全量编码 → `Z_k^target`（真相）。
4. **gaze 选一个局部观测 `o*`，注入后让 `Z_k^prior` 最贴近 `Z_k^target`**。
   "最大降低隐向量差别处" = **最 surprise / 最大信息增益处**，二者同义。

本质是**自蒸馏**：全帧 target 当老师，"想象 + 一次廉价扫视"当学生，学"用最少的看复现全看才能得到的修正"。
因 gaze 可导（STN），且训练期手握完整帧，可直接**算出最优 glance 当监督标签**教注视策略——把硬注意力从高方差 RL 降为可微监督。

### 5.2 三道防退化闸（缺一即坍缩成 trivial 零 loss）

若潜向量坍缩成常数，则 prior 恒等于 target、差别恒为 0、不用看、loss=0——却什么也没学。挡住它需要：

1. **SIGReg 防坍缩**——否则上述零 loss 退化解即全局最优。弃 Mamba 递归凸更新后，此项不是可选而是**承重**。见 §7 待建。
2. **target 必须 stop-grad（I8）**——否则"减小差别"可靠**移动靶子**（让 encoder 把 target 变得好预测）白嫖，仍是坍缩。
3. **观测必须有代价 `cost(obs)`**——否则"最大降低差别"的最优解是**全看**，退回稠密感知。这一项才逼出"偶尔扫一眼"，**是区别第 1 条区别于全量方案的全部**。

### 5.3 损失项草案

```
L = L_pred + λ·L_sigreg + β·L_inv + γ·L_obs_cost
```

- **L_pred**（预测）：`Z^prior` 逼近 **stop-grad** 的 `Z^target`，在**潜空间**算（JEPA 式，不解码回像素）。
  须**多步 rollout**（非单步），按精度加权（Gaussian NLL）：`Σ_i [ (Δz_i)²/σ_i² + log σ_i² ]`——可控维退化为 MSE 要求准，不可控维只求标定。
- **L_sigreg**（防坍缩）：把 `Z` 分布钉到各向同性高斯（Epps-Pulley sliced 检验）。
- **L_inv**（逆动力学，待建）：`InvDyn(Z_t, Z_{t+1}) → a_t`，给可控闸门 `c` 接地。
- **L_obs_cost**（注视/观测预算）：稀疏性来源；没有它设计在数学上不可能稀疏。
- 运行时（非预训练）：完美序列 goal-cost，System-2 想象去够。

### 5.4 两阶段调度（拆解鸡生蛋）

主动感知是 bootstrap，从零同时训"世界模型"和"注视策略"才有循环依赖。**分阶段即可断开**：

- **阶段一**：全观测、每帧全量编码、纯 LeWM 式预测 + SIGReg，先把**动力学与表征**训出来。此段**无鸡生蛋**。Z 用全编码初始化（有信息锚点）。
- **阶段二**：冻住（或慢更新）阶段一当**固定老师**，再蒸馏"稀疏 gaze + 修正"。**蛋已存在**，循环解除。
  随机初始化课程放这里：**偶尔**从随机/损坏的 `Z` 出发，专训"扫描把它拉回正确"（摊销迭代推断），顺带量化纠正回路强度。**不要**当阶段一默认——它与稀疏读取要求的"少纠正次数"打架。

### 5.5 收敛前提（为什么这样能学对，最易被无意违反）

LeWM 的"严格时序"是便利拐杖，免费给了几样东西；改成时空标签 + 稀疏后，必须**显式补回**：

1. **稠密无歧义 target** → 阶段一保持全观测（稀疏只在阶段二蒸馏）。预测的对象（Z）与监督的对象（稀疏读出）一旦解耦 + 观测稀疏，监督即欠定；稠密阶段一是定盘星。
2. **因果方向（arrow of time）** → **未来永远只当 target，绝不当 input**。时间编码只有两个合法用途：给**过去观测**打"几帧前发生"、给**预测查询**指定 horizon（`τ` token）；把待预测的未来事件带未来戳塞进输入 = 信息泄漏，学成抄答案。
3. **钉住欠定的稀疏 Z** → 长程多步一致性 + SIGReg（故 `L_pred` 须多步）。单步稀疏观测定不住 Z，但"对整条轨迹所有稀疏事件都自洽"像 SLAM 一样把状态逼到唯一。`WorldProbeDecoder` 验收。
4. **真动作喂 dynamics** → 训练 predict 吃**实际执行过的真动作**，不吃模型自己预测的动作（否则模型挑"好预测的动作"自欺）。动作规划头是 `Z` 上的**读出头**，梯度不在同一次前向回流去喂 dynamics。

**动态充分性**：单帧无速度（信息论事实，速度是时间导数）。要"相当于多帧动态"的不是单帧编码，而是 `Z`（belief state = 历史积分）。两个注入口：① 编码端开 **2–3 帧小窗**（中央凹裁"同区域跨帧的时空管"，保住局部运动），② 递归 `Z` 长程积分。冷启动单帧测不出速度——有界（再扫一眼即估出）且 `FiLM(g)` 任务先验可对冲（如"音符向下落"是可学游戏常量）。

### 5.6 观测策略（gaze + wake）的训练：别端到端闭环

模型自己输出**看哪里**（`gaze`）和**多久后再看**（`wake`/`T_wake`），下一步喂回自己——观测建议由上一次推理决定，是个**闭环**。端到端从零训这个闭环很麻烦：观测策略决定看到什么数据 → 数据又反过来训模型，是个**非平稳、含离散决策（何时睁眼）的控制问题，本质是 RL**。

**解法：不闭环训，拆成自监督 oracle 模仿**（与 §5.4 两阶段一致）：

- **阶段一**：观测策略**不参与**——用**外部固定/随机**的观测时刻与位置（全观测或随机裁剪），先把动力学训出来。无闭环、数据近似 i.i.d.。
- **阶段二**：世界模型冻成老师，gaze/wake 用**从自身预测误差算出的 oracle 当自监督标签**（训练期手握完整未来，oracle 可离线精确算）：
  - **看哪里（gaze）**：oracle = 预测误差热力图的峰值（哪最错就看哪）。正好**接管现有 `PeripheralVision.saliency` 头**——用误差热图监督它。可导模仿，非 RL。
  - **何时看（wake）**：oracle = 想象 rollout 与真相**首次发散超 ε 的时刻**（完整未来已知，可精确算）。把 `T_wake` 回归到它——离散的"何时睁眼"变成回归，**绕开 RL**。
- **可选最后一步**：两件都跑通后，小学习率轻度闭环微调，且仅在必要时。

oracle 全部来自"模型自身预测误差 vs 完整帧"，**不引入人工标注**——观测策略训练仍是**自监督**，只是把 RL 难题降成可微模仿。

## 6. 数值不变量映射（I1–I8）

| 机制 | 满足 |
|---|---|
| `SlotBinder` / `GatedResidual` 门控残差 | I5（增益受限、非扩张） |

| `crop_fovea` 的 affine_grid/grid_sample 用 fp32 | I4 |
| 感知/脑内全程 GroupNorm/LayerNorm，无 BatchNorm | I7（递归路径稳） |
| target stop-grad、不取消 detach | I8 |
| SIGReg、逆动力学等统计项只进 loss、不进前向 | I6 精神 |

## 8. 证伪探针

[net/world_probe.py](../net/world_probe.py) 的 `WorldProbeDecoder` 是一根**证伪探针**，不是世界模型本体：
刻意保持极薄（单隐层），若像素被遮挡时仍能从**聚合后的** `Z`（全体残差潜向量读出，非某个固定 slot）读出音符真实位置/轨道/颜色，
则读得准只能归功于 **`Z` 本身确实存了世界**（且已积分出速度），而非探针自己脑补。这是检验"脑内世界"是否成立的核心仪器。

## 9. 当前探针支架的数据流（passive prediction，**非完整系统**）

[train_probe.py](../train_probe.py) 是一个**最小有监督被动预测测试**，只验证"`Z` 能否存+预测一颗落音符"，**不是**本文设计的完整系统。

**每步输入** `model(img, Z, h, a_raw, dt, g_prev)`：

| 输入 | 形状 | 是什么 | 来源 |
|---|---|---|---|
| `img` | [B,3,256,256] | 观测帧 | env.render()（train_blackout 时随机涂黑） |
| `Z` | [B,N,d] | 上一刻脑内世界 | 初始 0，之后 = 上步 `mu`(detach)，**滚动累积** |
| `h` | [B,1,d] | 假说 token | 初始 0，之后 = 上步 `h_next` |
| `a_raw` | [B,t_hist,n_keys] | 动作历史 | **永远全 0** |
| `dt` | [B] | 预测跨度 | 恒定 0.05s |
| `g_prev` | 3×[B] | 上次注视 | 初始 (0,0,0.5)，之后 = 上步 `gaze` |

**输出**：世界信念 `Z_out`/`mu`/`sigma`（[B,N,d] **抽象潜向量**，非位置）+ `exist_p` + 决策头（`action`/`action_plan`/`gaze`/`T_wake`/`saliency`）。潜向量经 `WorldProbeDecoder` 才解码成人可读的 y/track/color。

**监督**：`loss_p`=探针解 `Z_out[:,0]`→逼近当前 GT；`loss_f`=探针解 `mu[:,0]`→逼近**下一帧** GT。GT 来自 `env.get_tracer_state()`。

**⚠️ 重大缺口**：`a_raw` 恒为 0 → **动作/可控性完全没测**。当前只证了被动预测，"动作有什么效果"这一项目核心**尚未进入测试**。

## 10. 实证记录（2026-06，CUDA RTX 3070）

**指标方法学（血泪教训）**：
- **`skill_occ`（固定盲区）是污染指标**——"看不见就猜固定盲区中点"无需记忆即可击败"冻结"。实测 **no_memory 也拿 +0.22**。**弃用**，除非随机化盲区位置。
- **唯一干净判据 = 开环全黑曲线 + memory/no_memory 对照**（全黑骗不了位置先验）。开环单轨迹方差大，须 **B=256 + 多 seed 去噪**。

**关键结果**：
- **架构有容量**（含外部修复：PeripheralVision 用 `PositionalEmbed`+cross-attn query 恢复空间定位、MLP/主干补 `LayerNorm` 稳尺度 I5/I7、eval off-by-one 对齐）：memory 模型 vis 11px；**开环 @16=25px vs no_memory 43px、@30=71px vs 124px** → **递归 Z 确实做（部分）前向模拟**，这是真世界模型的干净证据。
- **训练期遮挡有效**：`train_blackout`（随机全屏涂黑逼预测）把开环发散从 @16≈99px 压到 ≈22px。**训练期遮挡是已验证的核心配方**。
- **但前向模拟是部分的**：撑 ~16 步后衰减（@30 仍 71px）。

**教训**：① "低 loss ≠ 世界模型"，必须用抗作弊指标；② 观测永远可得 → 模型抄"读像素"近路、记忆萎缩 → 用训练期遮挡逼预测；③ env 已 GPU 化但 render 的 16-音符 Python 循环是瓶颈（B=256 吞吐 ~2× 于 B=32）；④ 可视化查看器 [view_env.py](../view_env.py) 已建（导出 env 视频/帧网格）。

## 11. 动作条件世界模型（2026-06 CUDA 已验证）

**项目核心"动作在场景里有什么效果"首次跑通**（此前 `a_raw=0` 完全未测）。

**测试床** [utils/rhythm_action_env.py](../utils/rhythm_action_env.py) `RhythmActionEnv`：Phigros 式**扁条音符**、**多音符**（每 lane 均值 3.3、全屏 ~13）、各音符**随机速度**（60–160px/s）、**先天规则:音符在判定线 ±60ms 内被按对键 → 消失（命中）**、**同 lane 相邻到线时间差 ≥100ms**。训练 [train_action_world.py](../train_action_world.py)：喂真实（探索性随机）动作进 `a_raw`，模型条件于动作预测下一帧每 lane (y, present, hittable)。

**结果（密集多音符）**：模型学到"按→消除"因果——
- **决策点准确率 DecAcc 94.3%**（随机基线 50%；仅在"当前可命中"lane 上阈值化预测 vs 真值，含"按了该消失/没按该留下"）。这是**事后读出的反馈，不是 loss**（符合设计意图）。
- 判别 Gap（未按−按掉）+0.55；动作打乱敏感度 4.5×（打乱动作 → 命中 lane 的 hittable 预测变差 4.5 倍）。
- `HitAcc 0.998` 不可当主数（被低基率抬高）。

**两把解锁钥匙**（缺一不可）：
1. **截断 BPTT（k=4）**：每步 detach 断掉跨时间梯度 → 动作时序因果学不了；BPTT 让 Gap 从 0 爬起。
2. **结构化 per-lane 读出**：`mu.mean(slot)` 把单 lane 动作效果稀释进 1/N → 读不出；改用 **n_lane 个 lane-query 对 slot 做 cross-attention**（同 PeripheralVision 模式），Gap 0.14→0.65。**戒律:多实体读出必须结构化 query、不能池化**（印证 §1 残差 slot 戒律）。
3. 配套：聚焦损失（只罚"当前可命中"lane，破易多数淹没动作信号）；shuffle 敏感度须**聚焦命中 lane**测，否则被位置/空 lane 稀释成假阴性（曾把真实 +3.3× 冲淡成 +0.04）。
4. 多音符回归后 `present` 浑（命中后下一颗顶上）→ 动作效果测 **`hittable`**；**100ms 间隔正好保证下一颗不在 ±60ms 窗内**，hittable 命中后干净归 0。

## 12. 预训练路径（规划）

rhythm env 是**验证用玩具**；真正的预训练将用**大量"带动作的游戏图像/视频"做无监督/自监督训练**：
- **数据 = 游戏画面流 + 对应动作流（按键/鼠标/MosPos）**。动作是录制的输入、未来帧是 target，**无需人工标注 → 自监督**。
- **目标**：在真实游戏录像上，以动作为条件预测未来（潜空间，JEPA 式），从"动作-画面配对"中学到动作的因果效果——即 §11 在 toy 上验证过的能力，放大到真实 footage。
- 沿用 §5 训练设计：predict-correct、SIGReg 防坍缩、训练期遮挡逼前向模拟、target stop-grad、多时延。动作头 / 可控闸门 c（§4）在此真实动作数据上才能充分训练。
